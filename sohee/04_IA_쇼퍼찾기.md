# IA 심화 — 쇼퍼찾기(전문가 매칭) 중심

> v1.0 · 2026-06-29 · 기반: [03_IA구조.md](03_IA구조.md) 보완
> 📍 **포지션:** 전체 MVP 순서는 *진단-우선* 그대로. 이 문서는 **쇼퍼찾기 갈래만 풀 디테일**로 펼친 것.
> ⚠️ **MVP의 쇼퍼찾기 = 페이크도어**(아래 §8). 그 아래 풀 마켓플레이스 구조는 **v2 타깃** — 다만 1기는 *수동 매칭(위저드 오브 오즈)*으로 코드 없이 검증 가능.

---

## 0. 쇼퍼찾기는 왜 '깊은' 도메인인가

진단(사이즈 번역)은 **단면 도구**라 화면이 직선이지만, 쇼퍼찾기는 **양면(사용자↔전문가) + 운영(중개)** 이 맞물리는 **마켓플레이스**라 IA가 3면으로 갈라진다.

```
        ┌──────────────── 매칭(쇼퍼찾기) ────────────────┐
[수요] 사용자 측  ──요청──▶   [엔진/중개]   ──전달──▶  [공급] 전문가 측
        프로필·요청·결제        매칭·정산·분쟁           수신·작성·정산
                                   ▲
                        [운영 어드민] 승인·중재·정산집행
```

핵심 다리(차별점): **체형진단의 '사이즈 번역 결과'가 매칭 요청에 자동 첨부**되어, 전문가가 *원격으로도 정확히* 고를 수 있게 한다(§5).

## 1. 쇼퍼찾기 도메인 사이트맵

```
쇼퍼찾기 (매칭)
│
├─ A. 사용자 측 (수요)  ── 👔 쇼퍼찾기 탭
│   ├─ S1 매칭 진입 · 서비스 유형 선택 ★
│   ├─ S2 전문가 목록 (정렬·필터)            ○v2
│   ├─ S3 전문가 프로필 (결과물·후기 검증)    ○v2
│   ├─ S4 매칭 요청 (유형별 폼 분기)          ○v2
│   ├─ S5 상담 / 메시지                       ○v2
│   ├─ S6 추천 / 예약 결과 (유형별)           ○v2
│   ├─ S7 결제 / 예약                         ○v2
│   └─ S8 완료 · 후기 · 평점                  ○v2
│
├─ E. 전문가 측 (공급)  ── 별도 진입면(전문가 앱)
│   ├─ E0 전문가 랜딩(혜택·수수료)
│   ├─ E1 지원/등록 → E2 심사 → E3 정산계좌·세금
│   ├─ E4 대시보드(요청·진행·가용 토글)
│   ├─ E5 매칭 요청 상세 ★(체형+사이즈 번역 결과 수신) · 수락/거절
│   ├─ E6 상담 → E7 추천/결과 작성(유형별)
│   └─ E8 상품관리 · E9 프로필/포트폴리오 · E10 리뷰 · E11 수익/정산
│
└─ O. 운영 어드민 (중개)
    ├─ O1 전문가 승인/심사 큐
    ├─ O2 분쟁·환불·신고 중재
    ├─ O3 정산 집행 · 수수료 정책
    ├─ O4 리뷰 관리
    └─ O5 매칭 성사율·GMV 대시보드
```

## 2. 사용자 측 매칭 플로우 (deep)

**진입점 2개:**
- 🔥 **Warm:** 체형진단 결과카드 → "전문가에게 받기"(사이즈 프로필 보유 = 따뜻한 리드)
- ❄️ **Cold:** 홈/탭에서 바로 진입(프로필 없음 → 진단 먼저 유도)

```
S1 진입·유형선택 → S2 목록 → S3 프로필 → S4 요청 → S5 상담 → S6 결과 → S7 결제 → S8 후기
   (MVP=여기서 페이크도어)        └──────────── v2 실동작 ────────────┘
```

| # | 화면 | 핵심 요소 | 비고 |
| --- | --- | --- | --- |
| **S1** | 매칭 진입 · 유형 선택 ★ | 서비스 유형(온라인 스타일링/동행 쇼핑/이미지 컨설팅) · 상황(소개팅·면접·결혼식·여행) · **(MVP)오픈 알림 신청 + 1기 모집** | 유형별 전환신호 ≥25% / 가입 게이트(트리거②) |
| **S2** | 전문가 목록 ○ | 정렬(매칭도순·평점·후기·가격) · 필터(유형 탭·상황·예산·지역·일정) · 전문가 카드(결과물·전문분야·평점·가격·**매칭도 뱃지**) | 매칭도 = 체형·상황 기반 |
| **S3** | 전문가 프로필 ○ | **결과물 갤러리(검증)** · 후기·평점 · 경력·전문분야·상황 태그 · 제공 서비스/가격/소요 · 가능 일정 | 얼굴·팔로워 ❌ / 결과물·후기 ✅ |
| **S4** | 매칭 요청 ○ | **공통:** 내 체형·사이즈 프로필 자동 첨부 + 상황·날짜 / **유형별 폼 분기**(§6) | 핸드오프(§5) |
| **S5** | 상담 / 메시지 ○ | 1:1 채팅 · 자리·예산·취향 컨텍스트 확보 | 스타일링의 본질 = 대화 |
| **S6** | 추천 / 예약 결과 ○ | 온라인: 코디·상품·구매링크 / 오프라인: 예약 확정·세션 안내·노트 | 유형별 결과물(§6) |
| **S7** | 결제 / 예약 ○ | 매칭 수수료·패키지 / (오프라인)예약금·세션비 | 과금 단위 = 상황 |
| **S8** | 완료 · 후기 ○ | 만족·후기·평점 · 재호출 의향 | 평판 피드백 루프 → S2 정렬에 반영 |

## 3. 전문가 측 (공급) — 전문가 앱

> 사용자앱 헤더 '전문가 지원' → E0 착지. 1기는 운영자가 수동 온보딩으로 갈음 가능.

```
E0 랜딩 → E1 지원/등록 → E2 심사/승인 → E3 정산계좌·세금
                                          ▼
E4 대시보드 ─ E5 요청상세 ★ ─ 수락 ─ E6 상담 ─ E7 결과작성
   (가용 토글)   (사이즈 번역 결과 수신)
[상시] E8 상품관리 · E9 프로필/포트폴리오 · E10 리뷰 · E11 수익/정산
```

| # | 화면 | 핵심 |
| --- | --- | --- |
| E0 | 전문가 랜딩 | 수수료·정산·기대 수익 · '1기 지원' CTA |
| E1 | 지원/등록 | 경력·전문분야·상황 강점·포트폴리오 |
| E2 | 심사/승인 대기 | 품질 게이트(↔ O1) |
| E3 | 정산계좌·세금 | 입금 계좌·사업자/원천징수 |
| E4 | 대시보드 | 들어온 요청·진행 중·**가용 상태 토글** |
| **E5** | **매칭 요청 상세 ★** | 사용자 체형 프로필 + **★사이즈 번역 결과 수신** · 수락/거절 | 
| E6 | 상담/메시지 | 추천 전 컨텍스트 대화 |
| E7 | 추천/결과 작성 | 온라인: 상품·코디·링크 / 오프라인: 예약·세션 노트 |
| E8 | 서비스 상품 관리 ★ | 제공 서비스 전시(메뉴판) · 유형별 범위·소요·가격 |
| E9 | 프로필/포트폴리오 | 노출 통제(= S3 원본) |
| E10 | 리뷰/평점 | 받은 리뷰 열람·응대 |
| E11 | 수익/정산 | 정산 내역 · 유형·상황별 수익 통계 |

## 4. 운영 어드민 (중개) — 매칭 관련

| # | 화면 | 핵심 |
| --- | --- | --- |
| O1 | 전문가 승인/심사 큐 | E2의 운영자 쪽 · 승인·반려 |
| O2 | 분쟁·환불·신고 중재 | 노쇼·불만족 환불 · 신고 대응 → **마켓플레이스 신뢰 핵심** |
| O3 | 정산 집행·수수료 정책 | 전문가별 지급 · 수수료율·유형별 요금 = **BM 손잡이** |
| O4 | 리뷰 관리 | 허위·악성 리뷰 조정 |
| O5 | 매칭 성사율·GMV 대시보드 | 거래액·성사율·활성 전문가 |

## 5. 핵심 다리 — 체형진단 → 쇼퍼찾기 핸드오프 ★

> 이게 우리가 일반 스타일링 매칭과 다른 **유일한 구조적 차별점**.

```
[체형진단 결과카드]                          [전문가 E5 요청상세]
  · 체형 프로필(캐릭터)        ──핸드오프──▶    · 고객 체형·핏 인사이트
  · 핏 인사이트(주의 부위)                       · ★브랜드별 추천 사이즈
  · 브랜드별 추천 사이즈                          → "원격으로도 정확히 고를 수 있다"
```

- **사용자 효과:** 요청 시 신체정보를 다시 안 적어도 됨(마찰↓) = 따뜻한 리드.
- **전문가 효과:** 줄자 없이도 사이즈 확신 → **온라인 스타일링(비대면)의 반품률↓**. 도구가 전문가 정확도를 올린다.
- **플랫폼 효과:** 이 데이터가 쌓일수록 매칭 품질↑ + v3 B2B 사이즈 API 자산.

## 6. 서비스 유형별 분기 매트릭스

> S4 요청 폼 · S6 결과물 · S7 과금이 **유형마다 다르다.** 한 흐름에 분기 3개.

| 항목 | 💻 온라인 스타일링 | 🛍️ 동행 쇼핑 | ✨ 이미지 컨설팅 |
| --- | --- | --- | --- |
| **요청 폼(S4)** | 취향·예산·선호 브랜드 | 일정·지역·만남 장소 | 목적(면접 등)·일정·지역 |
| **진행(S5·S6)** | 비대면 채팅·큐레이션 | 오프라인 동행 | 오프라인 세션 |
| **결과물(S6)** | 코디·상품·**구매링크** | 현장 구매·세션 노트 | 이미지 가이드·세션 노트 |
| **과금(S7)** | 매칭 수수료/패키지 | 예약금 + 세션비 | 세션비 |
| **사이즈 번역 활용** | ★최대(원격 구매 정확도) | 보조(현장 시착) | 보조(체형 이해) |

## 7. 상태(state) · 예외 흐름

- **요청 상태:** 대기 → `수락` → 진행 → 완료 / `거절`(→ 자동 재매칭 제안) / `만료`(무응답)
- **매칭 실패:** 후보 0명 → 오픈 알림 등록 · 조건 완화(예산·지역) 제안
- **취소/노쇼:** 환불 정책 분기(시점별) → O2 중재
- **분쟁:** 신고 → 운영 중재 → 환불/제재
- **품질 게이트:** 신규 전문가는 E2 승인 전 노출 ❌

## 8. MVP ↔ v2 경계 (페이크도어 → 실동작)

| | MVP (지금) | v2 (실동작) |
| --- | --- | --- |
| 사용자 측 | **S1 진입+유형선택 → 페이크도어 알림신청** ★ | S2~S8 풀 플로우 |
| 전문가 측 | 없음(운영자 수동 1기 모집) | 전문가 앱 E0~E11 |
| 운영 | 스프레드시트 수동 | 어드민 O1~O5 |
| 매칭 | **수동(위저드 오브 오즈):** 알림신청자 ↔ 1기 전문가를 사람이 카톡 연결 | 자동 매칭·결제·정산 |

> 👉 **MVP에서 검증할 것:** 유형별 전환신호(≥25%) + (수동 매칭으로) 실제 매칭 수요·전문가 수락률. **"따뜻한 리드면 낮은 수수료에도 전문가가 들어오는가"**(PRD 최대 가정)를 코드 0줄로 본다.
