## 프로젝트 한 줄
**fitting** — 평소 옷 착용감으로 **8체형 유형**을 진단해 **공유용 결과 카드**(도트 캐릭터)를 주고, 브랜드별 사이즈 번역 + 전문가 매칭으로 확장하는 서비스. 지금 폴더의 최근 작업은 **체형진단 결과 카드 UI**.

## 전체 서비스 그림 (제품 맥락)
- **핵심 가치(PRD):** 안 입어본 브랜드에서 "내 사이즈"를 몰라 망설이는 사람에게, **평소 잘 맞는 옷의 착용 경험**을 기준으로 **브랜드를 가로질러 사이즈를 번역**해 주는 AI 진단 카드. 진단으로 사용자·사이즈 데이터를 모아, 그 위에 **전문가(쇼퍼) 매칭**을 얹어 확장.
- **해자/차별점:** ① 몰 교차 ② 구매 이력 불필요(착용경험 기반) ③ 중립성(직접 안 팖).
- **MVP 2개:** ① AI 체형·사이즈 진단 카드 ② 전문가 추천(페이크도어로 전환신호 측정).
- **단계:** Now(진단=데이터 확보) → v2(전문가 실매칭 수익화) → v3(축적 데이터 = B2B 사이즈 API).
- **킬 메트릭:** 추천 사이즈 "실제와 맞다" 동의율 **≥ 65%**.
- **IA:** 4탭 — 홈 / 체형진단 / 쇼퍼찾기 / 마이 + **lazy 로그인**(결과 저장·매칭·마이 진입 시점에만). 쇼퍼찾기 = 전문가 둘러보기 + AI 전문가 매칭(페이크도어).
- **톤:** 미니멀 · 검정 + 포인트색 (05_스타일가이드).

## 8유형 (사이즈코리아 기반 + 우리 네이밍)
STR 시크 스트레이트(사각) · TRI 소프트 A라인(삼각) · INV 모던 V라인(역삼각) · HRG 엘레강스 X라인(모래시계) · BAL 이지 밸런스(표준) · DIA 소프트 다이아(마름모) · RND 코지 라운드(둥근) · TUB 슬릭 라인(튜브).
- 각 유형 = 코드 + 우리 이름 + 도트 실루엣 + 포인트색 1개 + 프로필 2줄 + 잘맞/피할 FIT(쉬운 말 2개) + FIT INSIGHT + 시그니처 해시태그 + 궁합.

## 확정된 핵심 결정 (되돌리지 말 것)
1. **16유형 → 8유형**으로. 입력이 빈약(옷 느낌)할수록 거친 분류가 신뢰↑. 16은 v2(사진·실측).
2. **사이즈코리아 8체형을 근거로, 표현은 우리 브랜드 이름/캐릭터.** (이름이 별로라 우리 것으로 표현)
3. 네이밍 = **V1(무드+라인)**, 카드 라벨은 **코드(STR·HRG…)**. "~체형 기반" 문구는 카드에서 뺌(문서에만).
4. 카드 = **블랙 배경 + 유형 포인트색 1개**(실루엣·코드·궁합·아이콘에만). 무채색 베이스.
5. 캐릭터 = **도트(픽셀) 패션 실루엣**(귀여운 블롭 ❌). 성별은 **머리**로: 여성=단발·갈색(#9a7350), 남성=짧은머리·검정(#2a2b30). 얼굴 skin #ecd9c9.
6. 카드 구조 = MY FIT(프로필 2줄) → FIT INSIGHT → 잘맞/피할 FIT → 🩷환상의 궁합(맨 아래). **구조 고정**(타입 바꿔도 박스 안 흔들림).
7. 잘맞/피할 FIT = **누구나 아는 쉬운 말**(테일러드·박시·로우라이즈 등 패션용어 금지), 각 2개·1줄. 방향성 팁은 FIT INSIGHT로.
8. 저장/공유 = 카드 상단 우측 **아이콘 버튼**(🔖🔗).
9. **8유형(스타일 식별)** 과 **사이즈 번역(브랜드별 사이즈)** 은 별개 엔진, 한 페이지 공존.

## 사용자 선호 (작업 방식)
- 만든 결과물은 **브라우저로 띄워서** 확인(이 환경은 Windows · `Start-Process`로 HTML 열기).
- 길게 설명보다 **바로 만들어 보여주기**. 피드백 짧게 주면 즉시 반영.
- "온점(.) 빼줘", "아이콘 덜어줘", "1cm만" 같은 **세밀한 톤 조정**을 자주 함 → 그대로 반영.
