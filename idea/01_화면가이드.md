# 화면 가이드 — 체형진단 기반 쇼퍼 매칭 (MVP)

> [00_기획산출물.md](00_기획산출물.md) 기반. 핵심 흐름은 **체형진단 → 결과 카드 → 쇼퍼 매칭 → 검증 → 만족**.
> ⚠️ MVP는 *수동(위저드 오브 오즈)* — 화면은 '고객이 보는 면'만 진짜고, 진단·매칭은 사람이 뒤에서 처리해도 됨.

---

## 0. 전체 화면 흐름

```
[S1 랜딩]
   │  "시작하기"
   ▼
[S2 상황 입력]  ← 어떤 순간(결혼식/여행)·예산·일정
   │
   ▼
[S3 체형진단 입력]  ← 사진/자가문진
   │
   ▼
[S4 체형 결과 카드] ───── 해결책 ① (우리 특화)
   │  "내 쇼퍼 찾기"
   ▼
[S5 쇼퍼 매칭 리스트] ─── 해결책 ②
   │  쇼퍼 선택
   ▼
[S6 쇼퍼 상세·검증] ───── 해결책 ③ (결과물·후기)
   │  "이 쇼퍼로 신청"
   ▼
[S7 신청·결제]  ← 단발 패키지 5~15만
   │
   ▼
[S8 추천 결과]  ← 쇼퍼가 고른 룩·구매처
   │
   ▼
[S9 만족 피드백] ──────── 북극성 (매칭 완료·만족)
```

---

## 1. 화면별 상세

### S1. 랜딩 / 가치 제안
- **목적:** "이게 뭐 하는 서비스인지" 3초 안에 이해 + 시작 유도
- **핵심 요소**
  - 헤드라인: *"중요한 순간, 실패 없이 입자 — 내 체형에 맞는 옷을 검증된 쇼퍼가 골라드려요"*
  - 서브: 결혼식·여행 등 '잘 입어야 하는 날' 강조 (순간 트리거)
  - CTA 버튼: **[시작하기]**
  - (신뢰) 쇼퍼 결과물 썸네일 몇 장 / 후기 한 줄
- **사용자 액션:** 시작하기 → S2

```
┌───────────────────────────┐
│   중요한 순간, 실패 없이 입자   │
│ 내 체형에 맞는 옷을           │
│ 검증된 쇼퍼가 골라드려요       │
│                           │
│   [ 시작하기 ]              │
│                           │
│  · 결과물 ▢ ▢ ▢  · 후기 ★★★★ │
└───────────────────────────┘
```

### S2. 상황 입력 (트리거)
- **목적:** "언제·얼마짜리" 순간인지 받아 매칭/추천의 맥락 확보
- **핵심 요소:** 이벤트 종류(결혼식 하객·여행·면접…), 날짜(D-day), 예산 범위, 한 줄 요청
- **사용자 액션:** 입력 완료 → S3
- **연결:** 문제정의의 "중요한 순간" / 리서치 #3

### S3. 체형진단 입력
- **목적:** '내 체형'을 판정할 최소 정보 수집 (입력은 가볍게)
- **핵심 요소**
  - 방식 A: **사진 업로드**(전신 1~2장) / 방식 B: **자가문진**(키·몸무게·어깨/허리/다리 비율 객관식)
  - "사진은 진단용으로만 쓰여요" 안내(신뢰)
- **사용자 액션:** 제출 → (MVP: 사람이 수동 판정) → S4
- **연결:** 해결책 ① · 페인 ①

### S4. 체형 결과 카드 ⭐ (우리 특화)
- **목적:** "나는 이런 체형"을 또렷이 알려줘 *진단의 가치*를 첫 경험
- **핵심 요소**
  - **내 체형 유형명**(예: "직선형 / 스트레이트") + 한 줄 설명
  - 잘 맞는 핏 / 피할 핏 요약 (2~3개씩)
  - 카드 저장·공유 버튼 (재방문·바이럴 씨앗)
  - CTA: **[내 쇼퍼 찾기]**
- **사용자 액션:** 쇼퍼 찾기 → S5
- **연결:** 해결책 ① · 시장조사의 'white space(체형진단)'

```
┌── 내 체형 카드 ──────────┐
│  유형: 스트레이트         │
│  "상체에 볼륨, 직선 실루엣" │
│  ✅ 잘 맞는 핏: 테일러드…   │
│  ⛔ 피할 핏: 프릴·퍼프…     │
│  [저장] [공유]            │
│  ─────────────────       │
│     [ 내 쇼퍼 찾기 ]      │
└─────────────────────────┘
```

### S5. 쇼퍼 매칭 리스트
- **목적:** 내 체형 프로필에 맞는 쇼퍼 후보 제시 (탐색의 막막함 해소)
- **핵심 요소**
  - 쇼퍼 카드 2~3개: 대표 결과물 썸네일 · 전문 체형/스타일 · 매칭 사유 한 줄("스트레이트 체형 하객룩 다수")
  - 정렬: '내 체형과 잘 맞는 순'
- **사용자 액션:** 쇼퍼 탭 → S6
- **연결:** 해결책 ② · 페인 ②

### S6. 쇼퍼 상세 · 검증 카드 ⭐
- **목적:** "이 쇼퍼 믿어도 되나"의 확신 (얼굴·팔로워 ❌ / 결과물·후기 ✅)
- **핵심 요소**
  - **Before/After 결과물 갤러리**(핵심)
  - **고객 후기**(별점·한 줄)
  - 전문 체형·이벤트 태그, 가격/소요일
  - CTA: **[이 쇼퍼로 신청]**
- **사용자 액션:** 신청 → S7
- **연결:** 해결책 ③ · 리서치 #4(결과물·후기로 고름)

### S7. 신청 · 결제
- **목적:** 단발 패키지 결제로 매칭 확정
- **핵심 요소:** 패키지 내용(진단+쇼퍼 추천) · 금액(5~15만) · 일정 확인 · 결제
- **사용자 액션:** 결제 완료 → S8
- **연결:** 리서치 #2·#3(진단/중요순간에 지갑 열림) · 과금 가설

### S8. 추천 결과 (쇼퍼 작업물)
- **목적:** 쇼퍼가 고른 "나에게 맞는 옷"을 받아본다 (단일/소수 추천)
- **핵심 요소:** 추천 룩 이미지 · 각 아이템 구매 링크 · 쇼퍼 코멘트("이 체형엔 이 핏")
- **사용자 액션:** 확인 → (구매) → S9
- **연결:** 해결책 종합

### S9. 만족 피드백 ⭐ (북극성 측정)
- **목적:** "이번엔 잘 입었다?"를 받아 **성공(매칭 완료·만족)**을 기록
- **핵심 요소:** 만족도(예/아니오 또는 별점) · 한 줄 후기 · "다음 이벤트에 또 부를래요?"(재호출 의향)
- **사용자 액션:** 제출 → 완료
- **연결:** **북극성 = "잘 입었다 만족한 매칭 완료 건수"** · 성공 기준(만족률 ≥70%·재호출)

## 2. 화면 ↔ 기획 정합표

| 화면 | 푸는 Pain | 해결책 | 측정 |
| --- | --- | --- | --- |
| S3·S4 체형진단·결과 | ① 내 체형 모름 | ① 진단(결과 카드) | 진단 완료율 |
| S5 쇼퍼 매칭 | ② 뭘 살지 막막 | ② 프로필 기반 매칭 | 매칭 진입율 |
| S6 검증 카드 | ③ 확신이 안 섐 | ③ 결과물·후기 검증 | 신청 전환율 |
| S7 결제 | — | 과금 | 결제율 |
| S9 만족 | (성과) | — | **북극성·만족률** |

## 3. MVP에서 진짜 만들 화면 vs 수동 처리

- **진짜 화면(고객이 봄):** S1 랜딩, S2 상황 입력, S3 진단 입력(폼), S4 결과 카드, S6 쇼퍼 검증 카드, S9 피드백
- **수동으로 때워도 되는 것(코드 0줄):**
  - S3→S4 진단 판정 = **사람이 직접** 보고 카드 작성
  - S5 매칭 = 쇼퍼 2~3명 중 **수동 배정**(자동 정렬 X)
  - S7 결제 = 계좌이체·간편결제 링크로 대체
  - S8 추천 = 쇼퍼가 **카톡/이메일**로 결과 전달해도 무방
- **만들지 말 것:** 회원 시스템 풀빌드, AI 자동 진단/추천, 쇼퍼 셀프 온보딩 — 전부 v2 이후

> 목표: "체형진단 받은 고객 → 검증된 쇼퍼 → 만족"이 **돈을 내고 일어나는가**를 가장 싸게 보는 것. 화면은 그 검증을 위한 최소 껍데기.
