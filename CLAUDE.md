# CLAUDE.md — 핏팅(Fitting) 프로젝트

> Claude Code가 매 작업 전에 읽는 파일. 이 프로젝트의 맥락·규칙·진행 방식을 담는다.

## 이 프로젝트가 무엇인가

새 브랜드 옷을 사고 싶어도 사이즈 확신이 없어 주저앉는 사람에게, **평소 입던 옷의 착용 경험을 기준으로 브랜드를 가로질러 내 사이즈를 번역**해주는 AI 진단 서비스. 이후 스타일리스트 매칭으로 확장하는 양면 플랫폼.

**한 문장 원칙:** 이 서비스의 가치는 **사이즈 번역 정확도 하나**에 걸려 있다. 화려한 UI보다 번역 정확도와 그 검증 루프를 항상 우선한다.

## 작업 시 반드시 지킬 것

1. **엔진 우선.** 데이터 → 엔진 → UI 순서(빌드 플랜 참조). UI를 먼저 완성하려 하지 말 것.
2. **사이즈 계산은 규칙 기반, 서술은 LLM.** 사이즈 추천 수치를 LLM에 맡기지 않는다(검증 불가). 여유 계산·실측 매칭은 결정론적 코드로, 체형 캐릭터·핏 인사이트 같은 자연어만 Claude API.
3. **피드백 로그를 처음부터 심는다.** 측정 못 하는 MVP는 MVP가 아니다. 진단 결과에는 항상 "맞았나?"를 수집하는 경로가 붙어야 한다.
4. **데이터 구조를 단방향으로 잠그지 않는다.** 사용자 착용 경험은 체형(B)뿐 아니라 브랜드 실측(A)도 교정한다(양방향 학습 여지).
5. **MVP 범위를 넘지 않는다.** IA의 ★만 실제 구현. ○는 목업/후속. 브랜드는 초기 N개만.

## 결정된 기술 스택 (출발점 — 변경 시 이 파일 갱신)

- **프레임워크:** Next.js (App Router) + TypeScript
- **배포:** Vercel
- **LLM:** Anthropic Claude API (자연어 인사이트·캐릭터 서술 전용)
- **DB:** 초기엔 파일 기반(JSON seed) → 검증 후 Postgres(예: Supabase/Neon)로 승격
- **스타일:** Tailwind CSS
- **상태:** 진단 플로우는 단일 멀티스텝(클라이언트 상태), 결과만 공유 가능한 별도 URL

> 위는 합리적 출발점이며 확정 전제는 아니다. Phase 0/1에서 더 단순한 선택(스크립트·로컬)이 맞으면 그쪽을 택하고 이 파일을 갱신한다.

## 폴더 구조 (제안 — 실제 생성 시 조정 가능)

```
/experiments/engine-feasibility/   # Phase 0: 타당성 스크립트 + 시드 + 결과
/data/seed/                        # Phase 1: 브랜드 실측 JSON seed
/src/lib/engine/                   # Phase 2: 역산·매칭·변환규칙 (규칙 기반)
/src/lib/llm/                      # Claude API 호출 (서술 생성)
/src/lib/schema/                   # 데이터 모델·타입
/src/app/                          # Phase 3: Next.js 화면 (진단 플로우)
/src/app/api/                      # 진단 API·피드백 로깅
/docs/                             # 기획 문서 (아래 인덱스)
```

## 문서 인덱스 (docs/)

작업 전 관련 문서를 읽는다.

- `fitting-1pager-prd.md` — 제품 정의·문제·페르소나·성공기준(킬 메트릭). **왜 만드는가.**
- `claude-code-kickoff.md` — 검증된 것/안 된 것, 데이터 구조, 진단 로직, 검증 루프, 수집 파이프라인. **구현 맥락.**
- `fitting-build-plan.md` — Phase 0~3 개발 순서. **무엇부터 만드는가.**
- `fitting-ia.md` / `fitting-ia.xlsx` — 전체 화면 정보구조(★ MVP / ○ 목업). **무슨 화면이 있는가.**
- `fitting-deck-content.md` — 발표 내용(서비스 서사). 맥락 보조.

## 현재 단계

**Phase 0 (엔진 타당성 실험)부터 시작.** 아직 Next.js 앱을 만들지 않는다.
먼저 `experiments/engine-feasibility/`에서 시드 데이터 3브랜드 + 역산 스크립트로 가설을 검증한다.

## 킬 메트릭 (성공/실패 기준)

- ★ 추천 사이즈가 "내가 실제 입는 사이즈와 맞다" 동의율 **≥ 65%**
- 보조: "안 사본 브랜드도 사보겠다" 의향 ≥ 50% / 전문가 유료 클릭 ≥ 25%
- 미달 시 엔진을 재설계하거나 멈춘다. 이 숫자가 모든 우선순위 판단의 기준.
