# 사이즈코리아 raw → CSV (빠른 재분석용)

원본 xlsx를 CSV로 변환. 매 분석마다 19MB xlsx 파싱 비용 제거(메인 시트 **3.2s → 0.03s, 113×**). 인코딩 `utf-8-sig`.
출처: 사이즈코리아 제8차 인체치수조사(2020~2024) / 국가기술표준원. 출처표기 의무.

## 파일

**① 통합 정규화 (분석 기본 — 이걸 쓰세요)**
| 파일 | 내용 |
|---|---|
| `통합_직접측정.csv` | **6,106명**(1-2차 5,092 + 3차 1,014) × 156항목. 두 코호트를 라벨 정렬해 병합, `cohort` 열로 출처 구분 |

정규화 내역:
- **성별 통일** M/F (남→M, 여→F)
- **라벨 정렬** — 코호트별 항목코드 차이 제거("041.가슴둘레"·"029.가슴둘레" → `가슴둘레`)로 두 코호트 병합.
- **단위 cm/kg** (길이 mm→cm). BMI·비만도·몸무게·나이는 변환 안 함.
- **센티넬 정제** — 9999 등 결측코드 제거(앞서 여성 어깨를 오염시킨 값 빠짐).

**② 체형 분류 (사람이 아니라 체형이 관측단위 → 별도 유지)**
| 파일 | 내용 |
|---|---|
| `표준체형.csv` | 표준체형 38개(1행=1체형). `derived/archetypes.json`의 소스 |
| `비만체형.csv` | 비만체형 32개(1행=1체형). 〃 |

> **코호트별 무손실 raw(mm) 분할본은 두지 않는다.** 그 데이터는 (a) `통합_직접측정.csv`에 정렬 병합돼 있고 (b) 원본 `data/body/raw/*.xlsx`에 그대로 보존된다. 원본 mm 코드 그대로가 필요하면 xlsx에서 재생성(생성 스크립트는 일회성 `scratchpad`).

## 커버리지 주의 (통합본)
- 둘레 핵심부위(가슴·젖가슴·허리·엉덩이) **100%**.
- **어깨가쪽사이길이 83%** — 3차 코호트가 미측정(이 항목은 1-2차만). 어깨 분석 시 `cohort=='1-2차'` 필터.
- BMI 16%(일부 연도만).
- 3D 자동측정·휠체어 시트는 의류 사이징 무관 → 변환 제외(xlsx에만 존재).

## 데이터 계층 (전체)
```
data/body/raw/*.xlsx      원본 소스(최종 진실, 무손실 mm 포함)
data/body/clean/*.csv     ① 통합 정규화(cm) + ② 체형 분류   ← 분석/재생성
data/body/derived/*.json  엔진 소비용 derived(회귀계수·분포·아키타입)
```
> derived JSON은 현재 1-2차만으로 생성됨. 통합본(6,106명)으로 재생성하면 표본 ↑.
