# 디자인 언어 — 핏팅

> **이 문서는 무엇?** 핏팅의 화면을 만들 때 지켜야 할 **색·글꼴·모양·말투 규칙**을 모았습니다. 디자이너가 아니어도 이 규칙만 따르면 화면이 한 덩어리처럼 보입니다.
>
> **용어 한 줄 풀이**
> - **토큰(token)**: 색·크기 같은 디자인 값을 이름표로 정해둔 것(예: "메인 초록 = `--green`"). 값을 한 곳에서 바꾸면 전체에 반영됩니다.
> - **잉크블랙**: 새까만 검정 대신 살짝 부드러운 검정(`#16140F`).
> - 실제 토큰 값은 **`web/tokens.css`** 한 파일에 있습니다(이게 정본). 화면은 이 값만 가져다 씁니다.

---

## 1. 한 줄 철학
**오프화이트 배경 + 딥그린 포인트 1가지 + 잉크블랙.** 차분하고 신뢰감 있게.

**왜 이렇게?** 우리 서비스의 핵심 가치는 "사이즈를 정확히 번역"하는 신뢰입니다. 화면이 화려하면 오히려 "AI가 대충 만든 느낌"이 나서 신뢰를 깎습니다. 그래서 **색을 아끼고, 글자 굵기와 여백으로 위계를 만듭니다.**

## 2. 5가지 원칙 (바꾸지 말 것)
1. **색은 딥그린 하나만.** 배경·글자·선은 무채색(회색 계열), 강조만 딥그린. 여러 색 섞지 않기.
2. **버튼은 검정(잉크블랙).** 주요 버튼은 초록이 아니라 검정. 초록은 '포인트'로만 써서 흔해지지 않게.
3. **글꼴은 Pretendard 하나.** 굵기(300~800)·자간·여백으로만 강약. (예외: 숫자는 §5)
4. **넓고 여유 있게.** 간격 넉넉히, 답답하지 않게.
5. **결과 카드만 예외** (아래 §4).

## 3. 색 (무엇을 어디에)
배경·표면·글자·선은 무채색으로, **딥그린은 "의미 있는 강조"에만** 씁니다.

```swatch
#F6F6F5 | 기본 배경 (오프화이트) | --bg
#FFFFFF | 카드 표면 (흰색) | --surface
#16140F | 본문·버튼 (잉크블랙) | --ink
#6E6B63 | 보조 텍스트 | --sub
#E6E5E1 | 경계선 | --line
#2E4A3B | 강조·활성·매칭도 (딥그린) | --green
#E7EFEA | 활성 배경 (연한 초록) | --green-soft
#B0553F | 경고·오류 | --warn
```

- 버튼 주요 = 검정 채움/흰 글자, 보조 = 흰 배경 + 얇은 테두리.
- 활성 상태(선택한 칩·탭·토글) = 딥그린.
- "맞음/성공" 신호도 딥그린 겸용(초록을 따로 안 만듦). "애매" = 회색, "틀림" = 경고색.

## 4. 결과 카드는 예외
체형진단 **결과 카드**만 **어두운 배경 + 8유형별 포인트색 1개**를 씁니다(카드가 유형마다 색이 달라지는 게 핵심이라). 여기엔 위의 딥그린 규칙을 적용하지 않습니다. — 8유형 색·데이터는 `web/data/bodytypes.json` 한 곳에서만 가져옵니다.

```swatch
#9DB8FF | STR · 시크 스트레이트 |
#76D6C6 | TRI · 소프트 A라인 |
#C2C7D0 | INV · 모던 V라인 |
#CF9BFF | HRG · 엘레강스 X라인 |
#8FD6A8 | BAL · 이지 밸런스 |
#FF9EC5 | DIA · 소프트 다이아 |
#FF9D7A | RND · 코지 라운드 |
#B6A8FF | TUB · 슬릭 라인 |
```

## 5. 글꼴(타이포)
- **본문·한글 = Pretendard.** 세리프(장식 있는 글꼴) 안 씀.
- **숫자(사이즈·cm·매칭도 %) = SUIT** 글꼴(`.num` 클래스). "정밀·측정"이 우리 정체성이라 숫자만 전용 글꼴로 또렷하게. (둘 다 한글 지원)
- 위계 예시: 큰 제목 44/37px·800, 소제목 24px·800, 본문 16px·450, 라벨 12px·800(영문 대문자). 큰 제목은 자간을 좁게(−.045em).

## 6. 모양(형태 토큰)
- 라운드: 버튼·입력 12px · 칩 20px · 카드 18px · 결과카드 24~30px.
- 테두리: 카드/선 `1px`(연회색), 입력/옵션 `1.5px`.
- 그림자는 아주 옅게. 컨테이너 폭은 앱 기준 넓게, 폼 화면은 읽기 편한 폭으로 중앙 정렬.

## 7. 컴포넌트 규칙 (빠른 참조)
| 컴포넌트 | 기본 | 선택/활성 |
|---|---|---|
| 버튼(주요) | 검정 채움·흰 글자 | hover 더 진한 검정 / 비활성 회색 |
| 버튼(보조) | 흰 배경·얇은 테두리 | hover 연한 회색 |
| 칩(필터) | 흰 배경·회색 | **딥그린 채움**·흰 글자 |
| 탭 | 회색 글자 | **딥그린** 글자 + 밑줄 |
| 토글 | 회색 | **딥그린** ON |
| 매칭도/정확도 뱃지 | — | **딥그린 pill**(연초록 배경) |

## 8. 말투(UX 라이팅)

> 핏팅의 말은 **친절한 스타일 친구**처럼 — 어렵지 않게, 판단하지 않게, 그리고 **일관되게**. 기본은 요체(~해요), 브랜드 언어는 '핏(fit)', 문장 속 이모지는 지양해요.

### 8-1. 세 가지 목소리
- **친근함** — 전문 용어 대신 누구나 아는 쉬운 말. 옆에서 골라주는 친구처럼.
- **명확함** — 한 문장에 한 메시지. 짧고 담백하게, 애매하지 않게.
- **신뢰** — 과장·호들갑 없이. 근거 있게 차분히 말해 믿음을 줘요.

### 8-2. 기본은 요체 (~해요)
격식체(~합니다)는 딱딱하고 반말은 가벼워요. 말끝은 **~해요 · ~예요 · ~드려요**, 명령은 **~해주세요** 처럼 부드럽게. 버튼은 짧은 동사구도 괜찮아요.
- **한 문장, 한 메시지** — 길면 끊어요.
- **문장 끝 온점(.)은 생략** — 마이크로카피는 마침표 없이 더 가볍고 친근해요.
- **호칭은 가볍게** — '고객님/회원님'보다 ~님 또는 생략. 우리를 낮춰 부르지 않아도 돼요.
- **버튼·라벨은 '~하기'로 통일** — *진단 시작하기·결과 저장하기·전문가에게 받기*. 같은 동작은 항상 같은 단어.

### 8-3. 브랜드 언어 '핏(fit)'
서비스 이름이 fitting인 만큼, 핵심 개념엔 **'핏'을** 일관되게 붙여 브랜드 자산으로 만들어요. (남용 금지 — 사이즈·체형·스타일 진짜 관련에만)

| 용어 | 뜻 | 예문 |
|---|---|---|
| **핏 진단** | 체형진단 (fit check) | "핏 진단 시작하기" |
| **핏 타입** | 8체형 유형 | "내 핏 타입은 시크 스트레이트예요" |
| **내 핏** | 내 체형·사이즈 프로필 | "내 핏 저장하기" |
| **핏 카드** | 결과 카드 | "핏 카드를 공유해보세요" |
| **핏 번역** | 브랜드 교차 사이즈 번역 | "브랜드별 핏 번역" |
| **핏 매칭 / 핏 매칭도** | 전문가 매칭 / 매칭 점수 | "핏 매칭도 98%" |
| **핏 인사이트** | 스타일 팁 (카드 내) | 카드 섹션 라벨 |
| **잘맞 핏 / 피할 핏** | 어울리는 / 피할 스타일 | 카드 내 섹션 |

- **한글 '핏'이 기본** — 본문·버튼은 한글 *핏*, 영문 **FIT** 표기는 결과 카드 같은 디자인 요소에만(카드 라벨은 MY FIT·FIT INSIGHT·잘맞/피할 FIT).
- **한 개념, 한 이름** — 같은 걸 여기선 '진단', 저기선 '분석'처럼 섞지 않기.
- **'핏메이트'(구 브랜드명) 지양** — 혼동돼요. 전문가는 *전문가* 또는 *쇼퍼*로.
- 결과 카드의 '잘맞/피할 FIT'은 각 1줄·쉬운 말. 방향 팁은 'FIT INSIGHT'로 몰아 씁니다.

### 8-4. 문장 속 이모지는 지양
이모지를 카피에 섞으면 통일감이 깨지고 톤이 들쭉날쭉해져요. **감정은 이모지가 아니라 글로** 전해요. 아이콘은 **기능 자리**(버튼·메뉴·상태 뱃지)에서만, 프로젝트에서 정해둔 세트로 통일해요. 카드 속 고정 심볼(예: 궁합 표시)은 카피가 아니라 디자인 컴포넌트 규칙으로 별도 관리.

**좋은 예 / 나쁜 예** (왼쪽처럼 쓰지 말고 오른쪽처럼)

```compare
당신의 체형은 역삼각형입니다 => 어깨가 넓은 편이라, 위는 여유 있게 입는 게 좋아요
Fit Type: STRAIGHT (사각) => 시크 스트레이트
사이즈 오류가 발생했습니다 => 사이즈를 못 찾았어요. 옷을 하나만 더 알려주실래요?
✨진단 완료! 지금 바로 확인하세요 🎉 => 진단이 끝났어요. 결과를 확인해 보세요
로우라이즈·크롭 기장 추천 => 밑위가 낮고 짧은 기장이 잘 어울려요
```

### 8-5. 상황별 마이크로카피 (권장 예문)

| 상황 | 이렇게 (권장) |
|---|---|
| 진단 시작 | "3분이면 내 핏 타입을 알 수 있어요" |
| 로딩 | "내 핏을 분석하고 있어요" |
| 결과 도착 | "내 핏 타입이 나왔어요" |
| 저장 유도 | "핏 카드를 저장하면 언제든 다시 볼 수 있어요" |
| 빈 상태 | "아직 맞는 전문가가 없어요. 1기 전문가를 모으고 있어요" |
| 에러 | "잠깐 문제가 생겼어요. 다시 시도해 주세요" |
| 성공 토스트 | "저장했어요" |

### 8-6. 쓰기 전 5초 체크
- [ ] **요체로 썼나요?** ~합니다·반말이 아니라 ~해요.
- [ ] **한 문장에 한 메시지인가요?** 길면 끊어요.
- [ ] **'핏' 용어를 일관되게 썼나요?** 같은 개념은 같은 이름.
- [ ] **문장에 이모지를 섞지 않았나요?** 감정은 글로.
- [ ] **판단·전문용어 없이 편안한가요?** 테일러드·박시 대신 쉬운 말.
- [ ] **문장 끝에 온점 없나요?** 마이크로카피는 마침표 없이.

## 9. 토큰은 어디에?
- 실제 값 = **`web/tokens.css`** (라이브에서 쓰는 정본). 색을 바꾸려면 이 파일의 변수만 고치면 전체 반영됩니다.
- 진단 화면은 `web/tokens.css` 규칙을 그대로 따르도록 `web/diag.css`가 맞춰져 있습니다.
