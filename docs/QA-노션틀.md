# 핏팅 QA — 노션 틀

> 사용법: 이 문서를 통째로 복사해서 노션 페이지에 붙여넣으면 표·체크박스가 그대로 변환됩니다.
> 3덩어리 구조 — **① IA/플로우(기준)** → **② 디자인 QA 체크리스트** → **③ 버그·이슈 트래커**
> QA의 핵심은 "만든 화면 vs 기준" 대조. 그래서 ①(기준 합의)이 먼저, 그 위에서 ②③를 돌립니다.

---

## ① IA / 플로우 — "정답(기준)" 합의

> 두 사람이 각자 그린 플로우를 여기서 하나로 합쳐 **기준으로 박제**합니다.
> 어긋나면 이 표를 먼저 고치고, 그 다음에 화면을 고칩니다. (기준이 흔들리면 QA가 무의미)

### 화면 목록 (13개)

| 트랙 | 화면 | 파일 | 담당 | 다음으로 이동 |
|---|---|---|---|---|
| 사용자 | 홈·허브(홈/쇼퍼찾기/마이) | index.html | | about, card, diag-basic, pro-signup |
| 사용자 | 서비스 소개·작동원리 | about.html | | diag-basic, index |
| 사용자 | 기본 정보 입력 | diag-basic.html | | diag-fit, index |
| 사용자 | 착용경험 보정 입력 | diag-fit.html | | diag-loading, result, index |
| 사용자 | 진단 로딩 | diag-loading.html | | result, index |
| 사용자 | 진단 결과 | result.html | | card, diag-fit, about, index |
| 사용자 | 결과 공유 카드 | card.html | | index |
| 쇼퍼 | 쇼퍼 지원(공급자) | pro.html | | pro-quote, card, index |
| 쇼퍼 | 쇼퍼 가입 | pro-signup.html | | pro, index |
| 쇼퍼 | 견적 요청 | pro-quote.html | | pro, index |
| 공통 | 약관 / 개인정보 / 데모초기화 | terms·privacy·_reset | | — |

### 핵심 플로우 (여기서 두 트랙이 만남 → 제일 잘 깨지는 곳)

```
[사용자 트랙]
 index(홈) → diag-basic → diag-fit → diag-loading → result → card(공유)
                                                        │
                                                        └→ card 링크로 초대 → index(초대받은 사람)

[쇼퍼 트랙]
 index(쇼퍼찾기 탭) → pro-signup(가입) → pro(지원) → pro-quote(견적요청)

[연결점 — 합의 필요]
 · index #shop 탭의 "인라인 견적"  ↔  pro-quote(견적요청)   ← 표면 관계 정리 필요
 · result → card 공유  ↔  card에 뷰어용 "나도 진단" CTA 없음  ← 흐름 끊김 확인
```

### 합의 체크 (두 사람이 같은 그림인지)
- [ ] 사용자 트랙 화면 순서·이동이 서로 같다
- [ ] 쇼퍼 트랙 화면 순서·이동이 서로 같다
- [ ] 연결점(쇼퍼찾기 탭 ↔ pro, 인라인견적 ↔ pro-quote)의 관계가 합의됐다
- [ ] 빠진 화면·중복 화면이 없다
- [ ] 각 화면 담당(누가 QA할지)이 정해졌다

## ② 디자인 QA 체크리스트

> 기준: 시안 · `tokens.css` · UX라이팅 가이드. 화면마다 아래 6개 결을 봅니다.
> 모션은 6개 중 하나일 뿐 — **일관성**이 2인 작업에서 제일 자주 깨집니다.

### 결(카테고리) 정의
| 결 | 무엇을 보나 |
|---|---|
| 일관성 | 두 화면에서 같은 버튼·카드·간격이 똑같이 생겼나 |
| 토큰 준수 | 색·폰트·간격이 tokens.css 변수대로인가, 하드코딩 없나 |
| 상태 | 빈 화면·로딩·에러·비활성·긴 텍스트일 때 안 깨지나 |
| 반응형 | 좁은 폭 / 넓은 폭에서 레이아웃 유지되나 |
| 모션·전환 | 화면 이동·눌림 피드백·애니메이션이 자연스러운가 |
| 카피·톤 | 요체 말투, '핏' 네이밍, 버튼 '~하기', 문장 끝 온점 생략 |

### 화면별 체크 (사용자 트랙)
**홈·허브 (index.html)**
- [ ] 일관성  · [ ] 토큰  · [ ] 상태(빈/로딩/에러)  · [ ] 반응형  · [ ] 모션  · [ ] 카피

**진단 3화면 (diag-basic / diag-fit / diag-loading)**
- [ ] 일관성  · [ ] 토큰  · [ ] 상태(입력 검증·긴 값)  · [ ] 반응형  · [ ] 모션  · [ ] 카피

**결과·공유 (result / card)**
- [ ] 일관성  · [ ] 토큰  · [ ] 상태(폴백·게이트)  · [ ] 반응형  · [ ] 모션  · [ ] 카피

### 화면별 체크 (쇼퍼 트랙)
**쇼퍼 (pro / pro-signup / pro-quote)**
- [ ] 일관성  · [ ] 토큰  · [ ] 상태(빈/에러)  · [ ] 반응형  · [ ] 모션  · [ ] 카피

### 크로스 체크 (트랙을 넘나드는 일관성 — 2인 작업 핵심)
- [ ] 사용자 화면과 쇼퍼 화면의 **버튼 모양·크기**가 같다
- [ ] 두 트랙의 **카드·입력폼·간격**이 같은 규칙을 쓴다
- [ ] 두 트랙의 **헤더·네비·뒤로가기**가 동일하게 동작한다
- [ ] 색·폰트가 양쪽 다 tokens.css만 참조 (하드코딩 0건)

## ③ 버그·이슈 트래커

> 노션에서 이 표 블록을 선택 → **"데이터베이스로 전환(Turn into database)"** 하면 필터·정렬 가능한 DB가 됩니다.
> 아래는 속성(컬럼) 설계 + 예시 행입니다.

### 속성(컬럼) 설계
| 속성 | 타입 | 값 |
|---|---|---|
| 제목 | 제목 | 이슈 한 줄 요약 |
| 트랙 | 선택 | 사용자 / 쇼퍼 / 공통 |
| 화면 | 선택 | index, diag-fit, pro … (화면 파일명) |
| 종류 | 선택 | 디자인 / 기능 / IA / 카피 |
| 심각도 | 선택 | 🔴높음 / 🟡보통 / ⚪낮음 |
| 상태 | 선택 | 열림 / 진행중 / 검토 / 닫힘 |
| 담당 | 사람 | |
| 비고 | 텍스트 | 재현 방법·스크린샷 |

### 예시 행
| 제목 | 트랙 | 화면 | 종류 | 심각도 | 상태 | 담당 | 비고 |
|---|---|---|---|---|---|---|---|
| 쇼퍼찾기 탭 버튼이 홈 버튼과 크기 다름 | 공통 | index | 디자인 | 🟡보통 | 열림 | | 크로스 일관성 |
| card에 뷰어용 '나도 진단' CTA 없음 | 사용자 | card | IA | 🟡보통 | 열림 | | 흐름 끊김 |
| 인라인 견적 ↔ 견적요청 관계 불명확 | 공통 | pro-quote | IA | 🔴높음 | 열림 | | 표면 관계 정리 |

## 진행 순서 (요약)
1. **① IA** — 각자 플로우 그려 오기 → 이 표에서 합쳐 기준 확정
2. **③ 트래커** 세팅 → DB로 전환
3. **② 디자인 QA** 한 화면씩 돌리며, 발견 즉시 ③에 이슈 등록
4. 이슈 닫으며 재점검
