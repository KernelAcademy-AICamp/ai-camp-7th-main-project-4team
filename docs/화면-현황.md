## 비고
- **데모 초기화** — sessionStorage 리셋 유틸.
- **서비스 소개·작동원리** — 소개→diag-basic 전환 CTA 있음.
- **관리자 · B2B API(3.D)** — IA 스텁 — API 고객·키·과금. v3.
- **관리자 · 사업현황(3.C)** — IA 스텁 — 운영 대시보드·제품 메트릭.
- **관리자 · 브랜드·제품 실측표(3.A.1)** — garments.json을 사이즈표(사이즈×부위 cm) 형태로 제품별 드릴다운. 생성물 뷰어(수정은 CSV+빌더).
- **관리자 로그인(구글+이메일2FA)** — 프로토타입 — 인증은 목업(실제 아님). 승격 시 Supabase Auth(구글)+이메일 OTP.
- **관리자 · 회원·거래(3.B)** — IA 스텁(빈 파일) — 전문가심사·회원·입찰·분쟁·정산·리뷰. v2.
- **관리자 · 내부운영(3.E)** — IA 스텁 — RBAC·감사로그·쿠폰. 후순위.
- **관리자 · 사이즈/데이터(3.A)** — 브랜드 실측(garments.json)+신체 사이즈(body-*) 실파일 브라우징. 커버리지·필터. 데이터는 실제, 로그인만 목업.
- **결과 공유 카드** — OG 공유 카드(소유자 아티팩트). card.js가 초대링크 index.html?from=CODE 생성 → 초대받은 사람은 홈으로 전환. 단 card 페이지 자체엔 뷰어용 '나도 진단' CTA 버튼은 없음.
- **기본 정보 입력** — 성별·키·몸무게. 성별 선택 시 평균값 프리필.
- **착용경험 보정 입력** — 앵커 브랜드·사이즈 완전 데이터화(#20). 엔진 역산 실연결은 상의(TOP) 기준.
- **진단 로딩** — 역산+매칭 계산 대기 → result로 이동.
- **홈·허브(탭 홈/쇼퍼찾기/마이)** — 단일 페이지에 탭 3섹션(#home/#shop/#my). 쇼퍼찾기=페이크도어.
- **견적 요청** — #17에서 개편. 수요자 인라인 견적(index #shop)과 표면 관계 정리 필요.
- **쇼퍼 지원(공급자)** — 공급자 진입.
- **진단 결과** — 상의만 실계산, 파생 카테고리는 선호핏 폴백/게이트.

### 상태 키별 생산자·소비자 (proto 스키마)
> 키 하나 = DB 엔티티/레코드 후보. 생산(씀)·소비(읽음) 화면이 스키마 경계·소유를 가늠하게 함.

| 상태 키 | 씀(생산) | 읽음(소비) |

| `fitting.admin` | admin-login | admin-login |
| `fitting.basic` | diag-basic | diag-fit, result |
