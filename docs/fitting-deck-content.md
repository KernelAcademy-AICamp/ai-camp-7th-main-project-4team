# 핏팅(Fitting) 발표자료 — 내용

## Slide 01 — 표지
핏팅
브랜드를 가로질러 '내 사이즈'를 번역한다
사이즈 고민을 넘어, 새로운 나를 입다

## Slide 02 — 문제 정의
사이즈 실패가 두려워 같은 브랜드만 시도하고
옷장엔 같은 옷들만 걸린다.

새 브랜드로 건너가려는 사람은 이 브랜드에서 내 사이즈가 뭔지를 몰라 망설인다.

무신사 등은 이미 사이즈를 추천하지만 자기 몰 안 + 구매 이력이 있을 때만 작동한다. 새 몰로 넘어가는 순간 다시 두려워진다.

## Slide 03 — 포지셔닝
기존 몰은 이미 추천한다. 단, 자기 울타리 안에서만.

- 몰 교차 — 특정 몰에 종속되지 않고 브랜드를 가로질러 번역한다.
- 이력 불필요 — 구매 이력이 아니라 평소 입던 옷의 착용 경험을 기준점으로.
- 중립성 — 직접 팔지 않으니 다른 브랜드로 건너가게 도울 동기가 있다.

→ 몰을 가로지르는 사이즈 데이터는 개별 플랫폼이 구조적으로 못 만든다. 이것이 해자.

## Slide 04 — 가설
새 옷 구매가 막히는 이유 (설문 결과)

- 40% 스타일 어울림 — 정답이 없다 → 전문가
- 38% 사이즈 적합 — 정답이 있다 → 도구

그래서 나눠 푼다.
두 축이 거의 반반. 하나만 풀면 절반의 사용자만 잡힌다.
성질이 다르니 푸는 주체도 다르게 — 사이즈는 도구가, 스타일은 전문가가.

## Slide 05 — 사용자 여정 (As-is → To-be)
As-is · 갇힌 옷장
1. 새 브랜드에 끌린다
2. "근데 사이즈 맞을까?"
3. 사이즈표·후기 봐도 확신 없음
4. 실패가 두려워 포기
5. 늘 사던 비슷한 옷만 산다

To-be · 건너가는 옷장
1. 새 브랜드에 끌린다
2. 핏팅 — "당신은 이 브랜드 L"
3. 사이즈 확신이 선다
4. 자신 있게 구매
5. 처음 보는 옷이 옷장에 걸린다

## Slide 06 — 제품·MVP
착용 경험으로 체형을 역산해, 브랜드별 내 사이즈를 번역한다.

기능 1. AI 체형·사이즈 진단
- 입력: 성별·키·몸무게·나이 + 착용 경험 보정 + 선호핏
- 출력: ① 체형 프로필 ② 핏 인사이트 ③ 브랜드별 추천 사이즈

기능 2. 전문가 추천 (페이크도어)
- 버튼 클릭만 로깅해 유료 전환신호를 측정한다. 실제 매칭은 다음 단계 — 수요부터 확인.

Aha — "안 입어본 브랜드인데, 이 사이즈면 맞겠다"는 확신이 처음 드는 순간.

## Slide 07 — 비즈니스
데이터 확보 → 수익화 → 데이터 확장

- NOW · 진단 — 진단으로 사용자·사이즈 데이터 확보. 주요 브랜드 N개로 한정. (수익원: —)
- v2 · 매칭 — 전문가 실매칭으로 수익화. (수익원: 매칭 수수료 · 커머스 제휴)
- v3 · 확장 — 사이즈 API(B2B, 반품 절감) + 상황별 카테고리. (수익원: 데이터 API · 상황별 과금)

상황(소개팅·면접·결혼식·여행)은 페인·지불 의향이 모이는 지점 — 매칭의 과금 단위가 된다.

## Slide 08 — 검증
Kill Metric: 추천 사이즈가 "내가 실제 입는 사이즈와 맞다" 동의율 65% 이상.
아는 브랜드로 검증 → 실구매 없이 엔진 정확도를 직접 측정. 미달 시 엔진을 갈거나 멈춘다.

보조 지표
- 50% 이상 — '안 사본 브랜드도 사보겠다' 의향 (페인 해소)
- 25% 이상 — 유료 클릭률 (전환신호)
