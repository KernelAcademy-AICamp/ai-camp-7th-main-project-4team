# 핏팅 IA (Information Architecture) — 정본

3-사이드 마켓플레이스(**사용자 · 전문가 · 운영자**) + 사이즈 진단 엔진.
이 문서가 화면·URL 구조·역할·범위의 기준이다.

**범례(우선순위)**  ● MVP · ◐ 일부 MVP · ▷ v2 · ○ 선택/후순위 · ▢ 목업(화면만)
**SEO 색인**  🔍 색인(공개 콘텐츠) · 🔒 noindex(개인·앱 상태)
**구현 상태**  ✅ 구현·배포 · 🟡 목업 · ⬜ 미착수

---

## 0. 이 IA를 규정하는 핵심 결정

1. **매칭 = 입찰(역경매) + 지명 병행**
   - 오픈 요청: 사용자 요청 → 전문가들이 **입찰** → 사용자 **낙찰**
   - 지명 요청: 사용자가 특정 전문가에게 **직접** 요청 → 수락/거절
2. **진단은 동의 없이 이용**(서비스 제공 근거) · **엔진 개선만 opt-in**(결과화면). 진입 동의 게이트 없음 — *구 IA의 G.5 필수 게이트는 폐기.* 만 14세 확인은 opt-in에 접목.
3. **비회원도 수집**(진단 제공에 필요한 최소) · **lazy registration**(저장·매칭 시점에 가입).
4. **SEO**: 공개 콘텐츠(랜딩·소개·전문가·가이드)만 색인, 진단·결과·매칭·마이·어드민은 noindex.
5. **엔진 정책**: 사이즈 수치는 규칙 기반(검증 가능), 캐릭터·핏 인사이트 서술만 LLM.

## 1. 사용자 (User)

- **● 1. 랜딩** `/` 🔍 ✅
  - 서비스 소개 + 진단 시작 CTA · 상황별 전문가 리스트 · 작동 방식 3스텝
- **● 1.1 기본 정보 입력** `/diagnose` 🔒 ✅ — 성별·나이·키·몸무게·선호핏
  - **● 1.1.1 착용경험 보정 입력** `/diagnose/fit` 🔒 ✅ — 역산의 핵심
    - 입어본 옷 [브랜드·핏·사이즈] ×1~3 · 부위별 느낌(끼임/딱맞음/여유/큼)
    - 실측 데이터 없는 카테고리(하의·파생)는 **선호핏만** 받아 진단(구조적 예외)
    - **● 1.1.1.1 진단 로딩** 🔒 ✅
    - **● 1.1.1.2 결과 카드** `/result` 🔒 ✅
      - ① 8유형 체형 프로필(캐릭터, LLM 자리 🟡) ② 핏 인사이트 ③ **브랜드별 추천 사이즈(핏 지수%)** ④ 체형 측정(부위 스펙트럼) ⑤ 옷장 게이트(이어서 진단)
      - state: 저신뢰(0벌)→회귀 추정 카드 · 재진단/초기화 · 결과 저장→가입 유도
    - **● 1.1.1.3 피드백 수집** 🔒 ✅ — "맞았나?"(킬메트릭 ≥65%) + 엔진개선 opt-in(만14세 확인 접목)
    - **● 1.1.1.4 결과 → 전문가 선택** 🔒 — "스타일까지 전문가에게" CTA → 1.2/1.3
    - **○ 1.1.1.5 결과 공유** 🔒 — 카드 이미지/링크
    - **○ 1.1.1.6 공유 카드 뷰(수신자)** `/r/:shareId` 🔒 — 바이럴 루프: 친구 카드 요약 + "나도 진단하기"
- **● 1.2 전문가 둘러보기(쇼퍼찾기)** `/experts` 🔍 🟡
  - **● 1.2.1 검색**(이름·키워드) · **● 1.2.2 상황 필터**(소개팅/면접/하객…) · **● 1.2.3 정렬**(매칭도/평점/가격)
  - **● 1.2.4 전문가 목록** — state: 0명·무결과→"1기 모집·오픈 알림"
    - **● 1.2.4.1 전문가 카드**
  - **● 1.2.5 전문가 프로필** `/experts/:id` 🔍 🟡 — 전문분야·포트폴리오·**후기**·서비스 정보·요청하기
- **● 1.3 전문가 매칭 요청(스타일링)** `/match` 🔒 🟡 — 상황·예산·일정·한줄요청·체형 프로필 첨부
  - **● 1.3.6 매칭 요청 보내기** — 오픈(입찰) or 지명
  - **◐ 1.3.7 오픈 요청 → 입찰 수신** `/match/:id` 🔒 ⬜
  - **● 1.3.8 입찰 비교·낙찰** `/match/:id/bids` 🔒 ⬜ *(입찰 모델 신규)* — 들어온 입찰(가격·제안·전문가) 비교 → 선택
- **◐ 1.4 마이페이지** `/my` 🔒 🟡
  - ◐ 1.4A 프로필 기본정보 `/my/profile` — 신체·선호핏(진단서 자동)
  - **● 1.4B 내 진단 결과** `/my/diagnoses` — 체형 카드·브랜드 추천 · ▷ 진단 이력·비교 · 재진단/공유
  - ▷ 1.4C 코디 요청 내역 `/my/requests` — 요청·입찰·진행 상태
  - ▷ 1.4D 즐겨찾기 `/my/favorites` — 찜한 전문가
  - ◐ 1.4E 계정 관리 `/my/account` — 로그인 정보 · ▷ 알림 설정 · 약관/개인정보(푸터로도) · 탈퇴
- **○ 1.5 로그인/회원가입** `/login` 🔒 🟡 — lazy(저장·매칭 시점). 소셜(카카오·네이버·구글)+이메일. 약관·만14세 동의.
- **▷ 1.6 요청 & 매칭 라이프사이클** 🔒 ⬜ — 대기→(입찰/수락·거절)→진행→완료→(취소/분쟁)
- **▷ 1.7 대화(메시지)** 🔒 ⬜ — 수락/낙찰 후 개설 · 결과물 전달 · 일정 조율 · 신고
- **▷ 1.8 결제 & 정산(사용자)** 🔒 ⬜ — 에스크로: 결제→보관→완료 후 정산 · 환불 규칙 · 결제 내역/영수증
- **▷ 1.9 고객센터 / 문의** 🔒 ⬜ *(보강)* — 1:1 문의·신고·환불 요청 진입

  - ▢ 2.0.4 정산 계좌·세금 정보

## 3. 운영 어드민 (Admin)  — 🟡 목업 / ⬜ 미착수 · `/admin/*` 🔒

- **▢ 3.A 사이즈 수집 / 데이터** — *해자의 실행 도구*
  - ▢ 3.A.1 브랜드·상품 사이즈 DB(브랜드×핏×사이즈 실측·출처·신뢰도)
  - ▢ 3.A.2 수집 정확도 모니터링(피드백 로그·오차 분석·미표기 부위)
  - ▢ 3.A.3 번역 규칙/파라미터(EASE_BANDS·역산 튜닝)
  - ▷ 3.A.4 수집 파이프라인 상태(후기 스크래핑·신규 브랜드 커버리지)
- **▢ 3.B 회원 · 거래 운영**
  - ▢ 3.B.1 전문가 승인/심사(자격 게이트)
  - ▢ 3.B.2 회원 관리(조회·정지·차단)
  - ▢ 3.B.3 **입찰/매칭 모니터링·개입** *(입찰 모델 신규)* — 미응답·어뷰징·유찰 관리
  - ▢ 3.B.4 분쟁·환불·신고 처리
  - ▢ 3.B.5 정산 집행·수수료 정책
  - ▢ 3.B.6 리뷰 모더레이션
- **▢ 3.C 사업 현황**
  - ▢ 3.C.1 운영 대시보드(GMV·수수료 매출·매칭 성사율)
  - ▢ 3.C.2 제품 메트릭(진단 완료율·정확도 동의율·전환)
- **▢ 3.D B2B API 사업 (v3)** — 미표기 부위 검열추정 = 교차 데이터 해자
  - ▢ 3.D.1 API 고객(쇼핑몰) 관리 · 3.D.2 키 발급 · 3.D.3 사용량·과금·매출 · 3.D.4 API 소개(외부용)
- **○ 3.E 내부 운영(후순위)** — 권한/역할(RBAC)·감사 로그·프로모션/쿠폰
