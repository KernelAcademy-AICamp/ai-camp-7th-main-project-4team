# 통합 설계 정리 (Phase A)

> 두 작업물(`sohee/`, `sangmin/`)을 하나로 보기 위한 **정본 문서**.
> 방향: **디자인은 sohee 기준, sangmin 작업을 그 위에 이식.**
> substrate: **통합 HTML 프로토타입 먼저** (Next.js는 후속). 당면 목적: **내부 설계 정리.**
> 이 문서가 확정되면 그 다음이 실제 화면 이식(Phase C).

---

## 0. 작업물 성격 (한눈에)

| | sohee/ | sangmin/ |
|---|---|---|
| 역할 | 보이는 것 — 디자인·화면 셸 | 계산하는 것 — 엔진·데이터·플로우 로직 |
| 기술 | 정적 HTML(바닐라 JS) | TS 엔진 lib(동작) + 정적 HTML |
| 담당 | 결과카드·마이·전문가(쇼퍼찾기)·**디자인 컨셉** | 엔진·시드데이터·나머지 사용자 플로우 |
| 디자인 | **딥그린 정본** | Precision 블랙(→재스킨) |

핵심: 겹치는 코드가 거의 없고 **상보적**. 겹치는 지점은 딱 두 곳 → §3.

## 1. 디자인 토큰 단일화 (딥그린 = 정본)

출처: `sohee/디자인가이드.html` 의 `:root`. 이것을 **단일 출처(single source of truth)** 로 삼는다.

### 1.1 뉴트럴 베이스
```css
--bg:#F6F6F5;      /* Paper 배경(오프화이트) */
--surface:#FFFFFF; /* 카드 표면 */
--ink:#16140F;     /* 본문·버튼(순검정 대신) */
--ink-soft:#3A382F; --sub:#6E6B63; --sub2:#A6A29A;
--line:#E6E5E1; --line2:#DAD8D2; --soft:#EDEDEA;
--warn:#B0553F;    /* 경고 전용 */
```
### 1.2 딥그린 액센트 (유일한 포인트색)
```css
--green:#2E4A3B;       /* 메인 딥그린 — 강조 키워드 */
--green-mid:#5A7A68;   /* 보조 강조 */
--green-light:#9CC1AE; /* 다크 배경 위 강조 텍스트 */
--green-soft:#E7EFEA;  /* 활성 메뉴·뱃지 배경 */
```

### 1.3 변수명 정규화 (⚠️ 지금 3종이 섞여 있음)
| 현재 | 위치 | → 통일 |
|---|---|---|
| `--green` / `--green-soft` | 디자인가이드.html | **정본 이름** |
| `--accent` / `--accent-soft` | sohee 핏팅_전체화면.html | `--green*` 로 통일(별칭 제거) |
| `--ac` / `--ok` | sohee `_보관/`·구 파일 | 폐기(블루 테마) |
| `--accent:#0a0a0a`(블랙) | sangmin fitting.css | **딥그린으로 치환** |

규칙: 이식 시 모든 화면이 위 `--green*`/뉴트럴 토큰만 참조하도록 CSS 변수 1벌로 수렴.

### 1.4 예외 — 절대 건드리지 않음
- **결과카드**는 다크 배경(`#16171b`) + 8유형 시그니처색 1개가 정본(디자인 원칙 5의 명시적 예외). 딥그린으로 재스킨 대상 아님.
  - 8유형색: STR `#9db8ff` · TRI `#76d6c6` · INV `#c2c7d0` · HRG `#cf9bff` · BAL `#8fd6a8` · DIA `#ff9ec5` · RND `#ff9d7a` · TUB `#b6a8ff`

### 1.5 타이포
- **Pretendard 단일**(fallback Apple SD Gothic Neo → sans). 위계는 배경색 아닌 **굵기·자간·여백**.
- sangmin의 **SUIT(측정 숫자 tabular-nums)** 는 "정밀=엔진 정체성"이라 **수치 전용으로 흡수**(사이즈표·cm·매칭도 %). 본문은 Pretendard 유지.
- 모든 폰트 스택 **한글 지원 필수**.

### 1.6 형태 토큰
`--r-btn/-input:12px · --r-chip:30px · --r-card:18px · --r-modal:24px`(결과카드 30px). 버튼 주요=`--ink` 채움/흰글자, 보조=흰배경+`1.5px --line2`. 컨테이너 폭 1240px.

## 3. 겹침 정본 결정

### 3.1 결과 화면 (양쪽 다 있음)
- **결과카드(8유형) = sohee 정본.** `sohee/데모/결과카드_단독.html` 을 컴포넌트로. props: `type`, `gender`. clip-path 8종·도트 캐릭터·구조고정 CSS 완성됨.
- **카드 주변 레이아웃 = sohee 결과웹** (좌 카드 / 우 브랜드 사이즈표+검증).
- **표시되는 값 = sangmin 엔진 공급.** sangmin `06-result.html` 은 폐기가 아니라 **로직 기증**: 브랜드별 추천 사이즈표·4축 스펙트럼 막대·신뢰도(저/보통) 배너 로직을 sohee 레이아웃에 흡수.
- ⚠️ 8유형 데이터가 sohee 쪽 여러 HTML에 JS `T` 객체로 **중복 하드코딩** → `진단카드_8유형.json` 을 **single source**로 만들어 중복 제거.

### 3.2 전문가 도메인 (역할 분리 — 충돌 없음)
- **유저측 전환**(브릿지·페이크도어) = sangmin.
- **전문가 마켓·전문가 앱**(쇼퍼찾기·상품관리) = sohee(+sangmin e22).
- 유저 결과 → "체형 프로필 자동 첨부" 로 전문가 요청에 연결(IA의 핵심 다리).

### 3.3 위저드
- sohee 셸에도 진단 위저드가 있으나, **입력 스펙 정본은 sangmin 04**(스키마와 1:1). sohee 위저드는 디자인 참고용, 내용은 sangmin으로 대체.

## 4. 네이밍 통일
- 탭/UI 문자열 = **"쇼퍼찾기"**, 내부 개념/코드 = **"전문가(expert)"**. 혼용 금지.
- 8유형 코드(STR/HRG…) ↔ silhouette 클래스(straight/xline…) 매핑은 **데이터의 `silhouette` 필드**로만.
- 우선순위 뱃지 `★MVP / ◐최소 / ○v2` 규칙 유지(MVP 범위 판단 기준).
