# 문서 작성 규칙 (기여자·AI용)

> **왜 이 문서?** 핏팅 기획 문서는 `.md`가 **원본**이고, [문서.html](문서.html) 뷰어가 이를 **딥그린 스타일로 렌더**합니다.
> 뷰어에는 일반 마크다운을 넘어서는 **전용 블록 렌더러**가 있어서, md를 아래 규칙에 맞게 쓰면 정적 HTML 못지않은 리치 화면이 됩니다.
> **새 md를 만들거나 고칠 때 이 규칙에 맞게 가공하세요.** (사람이 읽는 건 HTML, 원본·git·AI 관리는 md — 단일 소스)

---

## 0. 파이프라인 (먼저 이해)
```
_tpl.html (뷰어 껍데기: 스타일 + 렌더러)  +  *.md (내용 원본)
        └──────────────  문서생성.py  ──────────────┘
                              ↓
                         문서.html (배포·열람용, 자동 생성물)
```
- **내용은 `.md`에서만** 고친다. `문서.html`은 자동 생성물이라 직접 고치지 않는다(재생성 시 사라짐).
- 스타일·렌더러를 바꾸려면 **`_tpl.html`**을 고친다.
- 무엇을 고쳤든 마지막에 **재생성**: `python3 docs/문서생성.py`
- 뷰어에 새 문서를 추가하려면 `문서생성.py`의 `DOCS` 목록에 한 줄 추가.

## 1. 전용 블록 — 언제 무엇을 쓰나

일반 마크다운으로 쓰면 밋밋한 것들은 아래 **펜스드 코드블록**으로 쓰면 컴포넌트로 렌더된다.
판단 기준은 "무엇을 보여주려는가"다.

| 보여주려는 것 | 쓰는 블록 | 대신 쓰지 말 것 |
|---|---|---|
| 색 팔레트·토큰 | ` ```swatch ` | 색 이름만 나열한 표 |
| 잘못된 예 → 고친 예 (Before/After) | ` ```compare ` | "나쁜 예: … / 좋은 예: …" 글머리 |
| 화면·기능의 계층 구조(IA), 우선순위 | ` ```iatree ` | 중첩 불릿 리스트 |
| 사용자 여정·플로우(단계·분기·되돌아가기) | ` ```flow ` | ASCII 아트 흐름도 |
| 그 외 대비·요약·매핑 | **표(table)** | — (표는 자동으로 카드 스타일로 렌더됨) |

일반 표/불릿/코드/인용구는 그대로 써도 딥그린 스타일이 자동 적용된다. **표는 둥근 카드 + 회색 헤더로 렌더**되니, 대비·매핑·빠른 참조는 표를 적극 활용.

## 2. `swatch` — 색 스와치 그리드
한 줄에 색 하나. `HEX | 용도 | 토큰` (토큰은 생략 가능). 자동으로 칩+정보 카드 그리드로 렌더.

````
```swatch
#F6F6F5 | 기본 배경 (오프화이트) | --bg
#2E4A3B | 강조·활성 (딥그린) | --green
#9DB8FF | STR · 시크 스트레이트 |
```
````
- 흰색 계열(`#FFFFFF`)은 칩에 자동으로 하단 경계선이 붙는다.
- 실제 토큰 값의 **정본은 `web/tokens.css`**. 스와치는 문서용 미러이니 값이 어긋나지 않게.

## 3. `compare` — Before/After 2단 카드
한 줄에 대비 하나. `나쁜 예 => 좋은 예`. 좌측(✗ 경고색)·우측(✓ 딥그린) 2단으로 렌더.

````
```compare
당신의 체형은 역삼각형입니다 => 어깨가 넓은 편이라, 위는 여유 있게 입는 게 좋아요
사이즈 오류가 발생했습니다 => 사이즈를 못 찾았어요. 옷을 하나만 더 알려주실래요?
```
````
- 구분자는 반드시 `=>`. UX 라이팅, 네이밍, 톤 예시에 쓴다.

## 4. `iatree` — 정보구조(IA) 트리
접었다 펼치는 트리 + 우선순위 필터(전체/MVP만/v2·이후). **들여쓰기 2칸 = 한 단계**.
한 줄 형식: `우선순위 | 번호 | 이름 | 비고`

| 우선순위 기호 | 뜻 |
|---|---|
| `★` | MVP |
| `◐` | MVP 최소 |
| `○` | v2 이후 |
| `목업` | 목업만 |

- 노드 아래에 **`·`로 시작하는 줄**은 그 노드의 세부 동작(액션)으로 붙는다.
- 예시는 [2_화면과흐름.md](2_화면과흐름.md) §1-1 참고.

````
```iatree
★ | 탭1 | 진단 | 메인 진입
  ★ | | 기본 정보 | 키·나이
    · 키는 세로 치수 prior로만 사용
  ◐ | | 착용 경험 | 기반=상의+하의
○ | 탭2 | 내 핏 | 프로필
```
````

## 5. `flow` — 사용자 흐름도
세로 여정도. **한 줄 = 요소 하나**, `타입 | 필드 | 필드…` 형식. 카드 사이 화살표는 자동으로 이어진다.
색 키워드(점·배지): `ink` `ac`(진단 핵심=초록) `shop`(쇼퍼=중간초록) `warn` `green`.

| 타입 | 형식 | 렌더 |
|---|---|---|
| `phase` | `phase \| 색 \| 🏠 제목 \| — 부제` | 단계 구분선(색 점) |
| `node` | `node \| 스텝 \| 제목 \| 태그 \| 설명` | 기본 카드 (스텝·태그 생략 가능) |
| `hero` | `hero \| …(node와 동일)` | 강조(다크) 카드 |
| `gate` | `gate \| 제목 \| 설명 \| 트리거` | 🔒 점선 게이트 |
| `split` | `split \| 실패제목 \| 실패설명 \| 되돌아감 :: 성공제목 \| 성공설명` | 2단 분기(`::`로 좌·우 구분) |
| `branch` | `branch \| 아이콘 \| 제목 \| 태그 \| 설명 \| 하단메모` | 연속 branch는 2×2 그리드로 묶임 |
| `loop` | `loop \| 텍스트` | ↻ 되돌아가기 메모 |
| `conn` | `conn \| 라벨` | 라벨 붙은 화살표(라벨 없으면 짧은 연결선) |
| `legend` | `legend \| 색 라벨 \| 색 라벨 …` | 범례(각 칸 = "색 라벨") |

- **태그**: `★`로 시작하면 MVP(초록 pill), 그 외(`후속` 등)는 회색 pill.
- **수식 필드**(node/branch 어디든): `@강조문`(초록 강조 줄) · `metric:텍스트`(📊 메트릭 pill) · `color:shop`(쇼퍼색 카드) · `lead`(branch 강조 테두리) · `arrow:텍스트`(branch 하단 화살표).
- 실제 예시는 [2_화면과흐름.md](2_화면과흐름.md) §9 참고.

````
```flow
legend | ac 진단 | warn 게이트
phase | ink | 🏠 홈 | — 입구
node |  | 홈 랜딩 | ★ | CTA 3초 전달
conn | 진단 시작
gate | 개인정보 동의 | 진단 진입 필수
loop | 거부 ✗ → 홈으로 (막다른 길 없음)
split | 실패 | 옷을 더 추가 | ⟶ 2-2로 :: 성공 | 결과 카드 ↓
hero | 2-4 | 결과 카드 | ★ | 체형 프로필 + 추천 사이즈
```
````
