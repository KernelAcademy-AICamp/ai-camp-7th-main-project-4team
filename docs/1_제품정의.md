# 핏팅(Fitting) — 제품 정의 (PRD + 발표 서사)

> **이 문서는 무엇?** 핏팅을 "왜, 누구를 위해, 무엇부터 만드나"로 설명한 제품 정의서입니다. 처음 읽는 사람이 방향을 잡는 문서예요.
>
> **자주 나오는 용어 한 줄 풀이**
> - **킬 메트릭(kill metric)**: 이 숫자가 안 나오면 서비스를 접거나 갈아엎는 '생사 기준' 지표. 우리는 "추천 사이즈가 맞다" 동의율 **≥ 65%**.
> - **페르소나**: 대표 사용자 한 명을 구체적 인물로 그린 것. **JTBD**: 그 사람이 "이걸 왜 쓰는가"를 상황·목적으로 적은 문장.
> - **North Star(북극성) 지표**: 팀이 장기적으로 키우려는 단 하나의 핵심 숫자. **Aha 모먼트**: 사용자가 "오, 좋다!"를 처음 느끼는 순간.
> - **페이크도어(fake door)**: 실제 기능은 안 만들고 버튼만 두어 "얼마나 누르나"로 수요를 떠보는 방법. **해자(moat)**: 경쟁자가 쉽게 못 따라오는 방어벽.

## 1. 한 줄 요약

새 브랜드 옷을 사고 싶어도 "내 사이즈가 맞을지" 몰라 주저앉는 사람에게, 평소 입던 옷의 착용 경험을 기준으로 **브랜드를 가로질러 내 사이즈를 번역해주는** AI 진단 카드. 진단으로 사용자·사이즈 데이터를 모아, 그 위에 스타일리스트 매칭을 얹어 확장한다.

> 표지 카피: *"브랜드를 가로질러 '내 사이즈'를 번역한다 — 사이즈 고민을 넘어, 새로운 나를 입다."*

## 2. 문제 정의

새 브랜드로 건너가려는 사람은 "이 브랜드에서 내 사이즈가 뭔지"를 몰라 망설인다. 무신사 등은 이미 사이즈 추천을 하지만 *자기 몰 안 + 구매 이력*에 갇혀 있어, 새 몰·새 브랜드로 넘어가는 순간엔 아무도 번역해주지 않는다. 결국 실패가 두려워 늘 사던 브랜드의 비슷한 옷만 다시 사게 된다.

> 차별점이자 해자: ① 몰 교차 ② 구매 이력 불필요(착용경험 기반) ③ 중립성 — 직접 팔지 않으니 건너가게 도울 동기가 있고, 몰을 가로지르는 사이즈 데이터는 개별 플랫폼이 구조적으로 못 만든다(무신사는 자기 울타리 안에 갇힘).

## 3. 가설 — 왜 사이즈를 도구로 푸는가

새 옷 구매가 막히는 이유(설문): **스타일 어울림 40% / 사이즈 적합 38%.** 두 축이 거의 반반이라 하나만 풀면 절반의 사용자만 잡힌다. 성질이 다르니 푸는 주체도 다르게 —

- **스타일**은 정답이 없다 → **전문가**가 푼다 (후속 가설: 스타일리스트 매칭).
- **사이즈**는 정답이 있다 → **도구**가 푼다 (지금의 심장: 번역 엔진).

## 5. 사용자 여정 (As-is → To-be)

| | As-is · 갇힌 옷장 | To-be · 건너가는 옷장 |
|---|---|---|
| 1 | 새 브랜드에 끌린다 | 새 브랜드에 끌린다 |
| 2 | "근데 사이즈 맞을까?" | 핏팅 — "당신은 이 브랜드 L" |
| 3 | 사이즈표·후기 봐도 확신 없음 | 사이즈 확신이 선다 |
| 4 | 실패가 두려워 포기 | 자신 있게 구매 |
| 5 | 늘 사던 비슷한 옷만 산다 | 처음 보는 옷이 옷장에 걸린다 |

## 6. North Star / Aha

- **North Star Metric:** 주간 유료 추천 채택 수. 본 MVP는 그 입구인 **사이즈 번역 정확도 + 유료 전환신호**를 검증한다.
- **Aha Moment:** 진단 카드에서 *"평소 입던 옷 기준으로, 한 번도 안 입어본 브랜드인데 이 사이즈면 맞겠다"*는 확신이 처음 드는 순간.

## 7. MVP 기능 (2개)

- **AI 체형·사이즈 진단 [LLM + 규칙]** — 입력: 성별·키·몸무게·나이 + *입어본 옷 착용경험 보정* + 선호핏. 출력 카드: ① 체형 프로필(캐릭터) ② 핏 인사이트 ③ **주요 브랜드별 추천 사이즈**.
- **전문가 추천 받기 (페이크도어)** — 버튼 클릭만 로깅해 유료 전환신호 측정(실매칭 X). 수요부터 확인.

## 8. 단계적 범위 · 비즈니스 (Now / Next)

*"데이터 확보 → 수익화 → 데이터 확장"의 사업 단계.*

- **Now(MVP) · 진단:** 진단으로 사용자·사이즈 데이터 확보(주요 브랜드 N개 한정). 수익원 —.
- **v2 · 매칭:** 전문가 실매칭으로 수익화 — 매칭 수수료(B2C)·커머스 제휴 커미션.
- **v3 · 확장:** 축적 데이터를 사이즈 API(B2B, 쇼핑몰 반품 절감)로 확장 + 상황별(소개팅·면접·결혼식·여행 등) 카테고리·오프라인·퍼스널컬러. 상황은 페인·지불 의향이 집중되는 지점이라 매칭의 과금 단위가 된다. 수익원 — 데이터 API·상황별 과금.

## 9. 성공 기준

- ★ **킬 메트릭** — 추천 사이즈가 "내가 실제 입는 사이즈와 맞다" 동의율 **≥ 65%** (아는 브랜드로 검증 → 실구매 없이 엔진 정확도 확인. 미달 시 엔진 교체/중단).
- 보조: "안 사본 브랜드도 사보겠다" 의향 ≥ 50%(페인 해소) / 유료 클릭률 ≥ 25%(전환신호).

## 10. 열린 질문

- 체형·사이즈 추정 로직(규칙 vs LLM vs 혼합) — *이 MVP가 킬 메트릭으로 답할 것.*
- 착용경험 보정 몇 벌·어떤 부위.
- 브랜드 실측 확보·정규화(차별점이자 최대 난관).
- 페이크도어 클릭의 허수 보정.
