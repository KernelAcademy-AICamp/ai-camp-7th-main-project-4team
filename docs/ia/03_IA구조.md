## 0. 내비게이션 모델

사용자 앱은 **하단 글로벌 내비 4탭**. 그 밖에 글로벌 페이지(약관 등)와 **완전히 별도인 진입면 2개**(전문가 앱·운영 어드민).

```
┌──────────────── 사용자 앱 (User) ────────────────┐
│   🏠 홈        🔍 체형진단      👔 쇼퍼찾기      👤 마이   │   ← 하단 4탭
└──────────────────────────────────────────────────┘
   · 위에 떠 있음: 동의 게이트(G.5) · 로그인 모달(lazy) · 약관 푸터(G.3/4)

[별도 진입면]  ② 전문가 앱(목업, 공급측)   ③ 운영 어드민(목업)
```

**표기:** ★ = MVP 실제 구현 · ○ = 목업/후속(v2~) · ◐ = MVP는 최소만

## 2. 🔍 체형진단 탭  ★ — mvp1 무료 퍼널 (제품의 얼굴)

진입 시 **개인정보 수집·이용 동의(G.5) 게이트** → 위저드 시작.

```
[동의] → 2-1 기본정보 → 2-2 착용경험 → 2-3 로딩 → 2-4 결과카드 ─┬─ 2-5 피드백
                                                                ├─ 2-6 공유
                                                                └─ 2-7 쇼퍼찾기 전환 → (탭3)
```

| # | 화면 | 핵심 요소 | 비고 |
| --- | --- | --- | --- |
| **2-1** | 기본 정보 입력 ★ | 성별·나이 / 키·몸무게 / 선호핏(스키니·슬림·레귤러·루즈·오버) | 진단 1단계 |
| **2-2** | 착용경험 보정 ★ | 입어본 옷 `[브랜드+핏+사이즈]` ×2~3벌 · 부위별 느낌(목·어깨·가슴·소매·배·총장 → 큼/여유/딱맞음/불편) · `[state]`진행표시+이전수정 · `[state]`'목록에 없음' 대안입력 | **역산 정확도 핵심** / 핏 다른 2~3벌 / 길어 이탈방지 |
| **2-3** | 진단 로딩 ★ | 엔진 처리 대기(체형 역산 + 매칭) | LLM 호출 시 수 초 |
| **2-4** | 결과 카드 ★ | ① 체형 프로필(캐릭터) ② 핏 인사이트 ③ **브랜드별 추천 사이즈** · `[state]`저신뢰→'옷 더 추가' · 재진단/수정 · **저장→가입 유도** | 제품의 얼굴 / 비회원=URL 임시보존→가입 시 영구 = **전환 트리거 ①** |
| **2-5** | 피드백 수집 ★ | '실제 내 사이즈와 맞나?' 동의/비동의 · '안 사본 브랜드도 사보겠다' 의향 | **킬 메트릭 ≥65%** |
| **2-6** | 결과 공유 ○ | 카드 이미지 / 링크 | 바이럴 후크 |
| **2-7** | 쇼퍼찾기 전환 ★ | '스타일까지 전문가에게 받기' CTA → 탭3 | **무료→유료 퍼널의 다리** |

## 6. 별도 진입면 ② — 전문가 앱 (목업, 공급측)

> 홈 헤더 '전문가 지원' → 전문가 랜딩 착지. v2 본격화.

```
안내/혜택(수수료·기대수익) → 지원/등록(경력·포트폴리오) → 심사/승인 대기 → 정산계좌·세금 등록
대시보드(요청·진행·가용토글)
  └ 매칭요청 상세 ★[사이즈 번역 결과 수신 = 차별점] → 고객 상담 → 추천/결과 작성(유형별)
서비스 상품 관리 ★(전시=사용자 메뉴판) · 프로필/포트폴리오(3-4 원본) · 리뷰/평점 · 수익/정산
```
- ★ 포인트: **2.1.1 요청상세에서 '사이즈 번역 결과'를 받는다** = 도구가 전문가 정확도를 올리는 차별점. **2.2 상품관리**는 사용자에게 보일 메뉴판이라 가볍게 우선.

## 8. 핵심 플로우 — 비회원 진단 → lazy 전환

```
홈 → [동의] → 진단(2-1~2-3) → 결과카드(2-4, 비회원=URL 임시보존)
                                   ├─ 피드백(2-5) ………………… 킬메트릭 ≥65%
                                   ├─ 저장하기 → ★가입 모달(트리거①) → 영구보존
                                   └─ 전문가에게(2-7) → 쇼퍼찾기 진입(3-1)
                                                          → 알림신청 → ★가입 게이트(트리거②)
                                                          → 완료(3-2)  …… 전환신호 ≥25%
```
- **가입은 탭이 아니라 트리거.** 진단은 비회원으로 끝까지 → 가치를 본 뒤(저장·매칭) 가입 요구 = 이탈 최소화.
