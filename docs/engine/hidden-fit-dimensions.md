# 숨은 핏 치수 레지스트리 — 맞춤 측정항목 × 기성복 실측표 차이 (후기 검증)

> **무엇인가:** 기성복 온라인 **실측표에는 없지만**, 실제 착용 시 예상과 달라 불편을 만드는 "숨은 치수·속성"의 목록.
> **어떻게 도출했나:** ① **연역(뼈대)** — 맞춤옷 주문서가 재는 신체 치수 − 기성복 실측표가 주는 의류 치수 = 숨은 치수 후보. ② **귀납(증거)** — 한국 쇼핑몰·커뮤니티 실제 후기(verbatim)로 각 항목의 *빈도·표현 신호*를 채움.
> **왜 필요한가:** 이 표의 ★ 칸이 곧 [[a-side-imputation-moat]]의 "미표기 병목부위 A축 검열추정" 대상이자 [[growth-engine-pain-discovery]]의 페인축 레지스트리. CLAUDE.md 원칙 6(카테고리 단위)·원칙 4(양방향 학습)와 직결.

## 핵심 정의

**축 정렬:** 맞춤옷 주문서 = **신체 실측(体)**, 기성복 실측표 = **의류 평면 실측(物)**. 둘의 차이가 곧 엔진 코어인 **여유량(ease)**.

> **숨은 치수 = 맞춤옷이 재는 신체 치수인데, 기성복이 대응하는 의류 치수를 안 주는 항목.** → 그 부위는 여유 계산 불가 → 예상 못 한 불편 발생 = 미표기 병목 = 해자 후보.

**제외 대상:** 실측표에 *있는* 숫자(총장·단면)의 단순 "크다/작다"는 이미 풀린 문제라 제외. **없어서 놀라는 지점**만 수집.

### 검증으로 드러난 핵심: "맞춤"은 두 층위다 (뼈대 재정의)

웹 검증 결과, "맞춤이 재는 치수"는 하나가 아니라 **두 층위**로 갈린다. 이게 A축 검열추정 난이도를 좌우한다:

- **(1) 전문 채촌(테일러 관측)** — 맞춤정장 재단사가 재는 리치 세트: 목둘레·어깨·상동(가슴)·중동(배)·화장·앞품·뒷품·진동(암홀)·허리·밑위·등길이 + **자세(굴신체/반신체/정체) 분류**. 근거: [스튜디오 더수트 채촌 FAQ](http://www.thesuit.co.kr/xe/board_Xwyr68/215), 자세 보정은 [특허 KR101182256B1](https://patents.google.com/patent/KR101182256B1/ko)로 확인. **단, 이 세트는 사람이 관측·분류하지 소비자가 입력하는 숫자가 아님.**
- **(2) 온라인 셀프 측정 폼(소비자 입력)** — 컴팩트 세트만: 상의 {목둘레·어깨너비·가슴둘레·허리둘레·소매길이·소매통·총장}, 하의 {허리·엉덩이·인심}. 근거: [젠틀리스트 셀프셔츠 폼](https://www.gentlist.co.kr/service/self_shirt.php), [유니클로 커스텀오더](https://faq-kr.uniqlo.com/pkb_Home_UQ_KR?id=kA0fR0000001Mhd).

**함의 1 — 시장 검증:** 온라인 맞춤 서비스는 이미 **"내가 잘 맞게 입는 기성옷을 평평히 놓고 재서 입력"**하는 경로를 표준으로 제공(젠틀리스트 "기성 셔츠 측정법", 유니클로 "기존 바지 인심"). = 핏팅의 **착용경험(기준옷) 번역 전제가 시장 관행으로 이미 존재**한다는 직접 근거.

**함의 2 — 추정 난이도 3등급:** 숨은 치수는 "얼마나 소싱 가능한가"로 나뉜다.
- **(A) 소비자 소싱 가능** = 온라인 셀프폼이 이미 받는 것(소매통·목둘레 등) → MVP 입력에 바로 넣을 수 있음.
- **(B) 전문 채촌만 아는 것** = 진동·화장·앞뒤품·중동·자세 → **A축(브랜드 실측) 검열추정으로 채워야 하는 진짜 해자 구간.**
- **(C) 양쪽 다 안 재는 공백** = 무릎둘레·종아리둘레 → 페인은 실재하나 어느 주문서에서도 소싱 불가, 추정 난이도 최상.

**레지스트리 스키마(엔진용):**

| 필드 | 의미 |
|---|---|

## 2. 하의

> 기성복 바지 실측표는 허리·엉덩이·허벅지단면·밑위·밑단까지 **비교적 잘 준다.** 그래서 상의보다 "치수 부재"는 적고, 남는 숨은 축은 **비율·곡선·동적 여유·소재**에 몰림.

## 6. 횡단 발견 (전 카테고리 공통)

1. **실측표 신뢰도 자체가 붕괴.** 표기 허리 73cm ↔ 실측 67cm(6cm 차), "정상 오차 1~3cm"에 법적 기준 없음 — [소비자가만드는신문](https://www.consumernews.co.kr/news/articleView.html?idxno=701073). "무신사 입점 도메 바지들 실측 정말 다틀려서" — eomisae. → **A축(브랜드 실측) 단독 의존 위험** → B축(착용경험 역산) 필요 근거(CLAUDE.md 원칙 4).
2. **사용자는 이미 실측을 못 믿고 "체형"으로 번역 중.** "기성복 66인데 어깨가 있는 편이라 L 주문 … 잘 맞아요" — jisudaum. = 이 서비스의 논리와 동일.
3. **모델컷이 단점을 은폐.** "상세사진에 팔을 다 몸쪽으로 붙이고 있어서 몰랐다"(암홀 과대) — 사이즈표+사진 **둘 다** 못 잡는 정보 공백.
4. **경쟁(무신사) AI 후기 요약이 카테고리별로 다른 축 사용**(바지=사이즈·핏·신축성 / 아우터=디자인·보온·소재) — [무신사 뉴스룸](https://newsroom.musinsa.com/newsroom-menu/2026-0212-1). CLAUDE.md 원칙 6(카테고리 단위 진단)의 시장 검증. 차별화 지점 = 무신사도 숫자화 못 한 **부위별 편차 역산·동적 여유**.
5. **대처가 하나같이 "한 사이즈 업" 또는 "수선"으로 수렴** → 숨은 치수를 제공하면 반품·수선 감소 여지 큼(패션 반품 사유 중 사이즈/핏 54%).

## 7. 엔진 반영 우선순위 (MVP 상의 기준)

후기 빈도 × 실측표 부재도로 볼 때 **상의에서 먼저 스키마에 넣을 숨은 치수:**

1. **진동(암홀) 깊이·둘레** — 최빈 페인, 레이어드/가동성 결정
2. **소매통(상완둘레)** — 소매길이와 독립 축
3. **어깨경사·봉제선 위치·드롭 정도** — 어깨너비 수치와 별개
4. **원단 두께·비침·신축률** — 같은 치수라도 여유 달라짐(세탁 후 시점 변화는 [[fitting 시점 버저닝]]과 연결)

이 4개가 [[a-side-imputation-moat]]의 A축 검열추정 1순위 대상. 사용자에게 "안에 뭘 입을지 / 팔뚝이 낀 적 있는지"를 물어 역산하면 [[growth-engine-pain-discovery]]의 페인→인사이트 루프로 되돌릴 수 있음.
