# 역산·추천 판단 로직 (규칙 기반)

> **무엇인가:** 착용경험 신호 → 체형 역산 → 사이즈 추천 → 경고까지의 **결정론적 판단 규칙**. 수치·판정은 전부 규칙 기반, 자연어(체형 캐릭터)만 LLM (원칙 2).
> **입력:** [../diagnosis-questions.md](../diagnosis-questions.md)의 `산출 데이터` 필드. **참조:** 숨은 치수·경고 근거 [../hidden-fit-dimensions.md](../hidden-fit-dimensions.md).
> **위치:** 이 폴더(`docs/engine/`)는 판단 로직 전용. 구현은 `/src/lib/engine/`.

## 0. 파이프라인

```
signals ──▶ [1] subtype 게이팅 ──▶ [2] 부위별 역산 ──▶ [3] 체형진단 카드 조립
                                                              │
        [7] 옷속성 경고 ◀── [5] 타겟 매칭·추천 ◀── [4] 병목 판정
                                                              │
                                          [6] A축 교정(양방향) · [8] 시점 게이트
```

기본 관계식: **여유(ease) = 의류 실측 − 신체 치수.** 경험이 주는 건 여유의 부호·크기 신호 → `신체 = 의류실측 ∓ 경험여유`.

---

## 1. `garment_subtype` 게이팅

성립 안 하는 신호는 역산에서 제외하고 prior로 대체.

| subtype | 무효화 신호 | 대체 |
|---|---|---|
| 반팔 | `sleeve_len` | 팔길이 ← `height` prior |
| 민소매 | `sleeve_len`, `bottleneck=팔뚝` | 암홀만 유효 |
| 반바지 | `hem_break` | 기장 ← `height` prior |
| 크롭(하의) | `hem_break` | — |

> 게이팅은 A축 실측 해석에도 적용(반팔 소매 스펙 ≠ 긴팔).

## 2. 부위별 역산 규칙 (신호 → 신체 추정)

δ = 튜닝 파라미터(Phase 0 캘리브레이션 대상).

| 신호 | 규칙 | 산출 신체값 |
|---|---|---|
| `global_offset` | 작음 `+δ` / 큼 `−δ` | 전역 기준선 |
| `shoulder_offset` | 넘어감=옷어깨>내어깨 → `어깨_body = 실측 − δ` · 안쪽 `+δ` · 딱 `≈` | 어깨너비 |
| `armhole_pressure` | 가끔=1·자주=2 → `요구진동 = 실측진동 + k·level` | 진동 요구치 (+ [6] A축 교정) |
| `chest_ease` | 낌 `가슴_body = 실측 + δ` · 남음 `−δ` | 가슴둘레 |
| `belly_ease` | 동일 방식 | 배둘레(중동) |
| `sleeve_len` (긴팔) | 짧음=팔 김 → `+δ` · 김 `−δ` | 팔길이 |
| `width_flag` | 당김 → 앞뒤품 부족 플래그 | 앞뒤품 |
| `waist_hip_drop` | 엉덩이낌 → 낙차 큼(엉덩이>표준비율) · 허리남음 → 낙차 큼(다른 방향) | 허리-엉덩이 낙차 |
| `hi_hip_flag` | 벌어짐 → 하이힙 볼륨 큼 | 앞판 여유 요구 |
| `back_rise_flag` / `front_rise_flag` | 파임/처짐 → 밑위 요구치 | 앞·뒤밑위 |
| `thigh_ease` | 낌 `+δ` · 남음 `−δ` | 허벅지 |
| `hem_break`(긴바지) | 끌림 → 기장 김 · 짧음 → 짧음 | 총장 보정 |

### 가슴×배 조합 판별 (체형 유형)
| `chest_ease` | `belly_ease` | 판정 |
|---|---|---|
| 맞음 | 낌 | 중동 볼륨형(윗배·복부) |
| 낌 | 맞음 | 상흉부형 |
| 남음 | 남음 | 루즈 선호 / 마른형 |
| 맞음 | 맞음 | 표준 |

## 3. 체형진단 카드 조립 (두 기반의 조립 산출물)

```jsonc
{
  "card_id": "…",
  "effective_date": "2026-07-01",   // 시점 스냅샷(불변·append-only, 원칙 7)
  "weight_version": 62,             // 체중 바뀌면 새 버전 append
  "upper": {                        // from 상의 진단
    "shoulder": { "width": "넓음", "slope": "각짐" },
    "chest_ease": "표준", "belly": "여유필요",
    "armhole_need": "큼", "arm_length": "표준",
    "confidence": 0.82
  },
  "lower": {                        // from 하의 진단
    "waist_hip_drop": "큼", "thigh": "발달",
    "front_rise": null, "back_rise_need": "큼",
    "confidence": 0.76
  },
  "completeness": { "upper": true, "lower": true },  // 파생 해금 게이트
  "character": "어깨 넓고 하체 볼륨형"   // ← LLM 서술(수치는 규칙기반)
}
```

- **수치·요구치 = 규칙 기반**, **character 한 줄만 LLM.**
- 상의만 완료 시 `upper`만 채움 → 상체계열 파생(아우터)만 부분 해금.
- 재진단(체중 변화)은 카드를 고치지 않고 새 `effective_date` 버전 append.

## 4. 병목 판정 (부위 충돌 해소)

부위별 요구가 충돌하면(어깨는 맞는데 진동 부족) **가장 크게 걸리는 부위(병목)를 기준으로 사이즈를 잡고 나머지는 경고.** 커뮤니티 휴리스틱("병목 부위에 맞추고 허리는 벨트")과 사용자 멘탈모델 일치.

```
bottleneck = argmax_zone( 요구치 − 해당사이즈 실측 )   // 가장 부족한 부위
추천사이즈 = 병목 부위를 만족하는 최소 사이즈
그 외 부위 = "여유/부족" 경고로 표시
```

## 5. 타겟 추천 규칙

타겟옷 T(브랜드·핏라인·사이즈군)의 A축 실측에 대해:

```
목표 = 신체 + target_ease           // 부위별
후보 = { s | 하드제약 만족: 진동_T(s) ≥ 요구진동, ... }
추천 = 후보 중 목표에 최근접(병목 우선, [4])
출력 = 추천 사이즈 + 부위별 예상 핏 + 경고
```

## 6. A축 교정 (양방향 학습, 원칙 4)

사용자 경험이 A축 실측/그레이딩과 어긋나면(스펙상 정사이즈인데 진동 자주 낌) → 그 브랜드·핏라인의 **미표기 병목부위(암홀·앞뒤품 등) 추정치를 갱신**(검열추정). B(체형)만이 아니라 A(브랜드 실측)도 교정. 규모 누적 시 [../hidden-fit-dimensions.md]의 ★(B등급) 칸이 데이터로 채워짐 = 해자.

## 7. 옷속성 경고 (판단이되 질문 아님)

A축 속성에서 파생, 사용자에게 묻지 않고 **결과에서 경고**:

| 속성(A축) | 경고 조건 | 문구 예 |
|---|---|---|
| 비침(색상·두께) | 밝은색·저GSM | "이 제품 비침 있음 — 이너 권장" |
| 신축(스판) | 논스판 + 타이트 추천 | "스판 없어 여유 빠듯할 수 있음" |
| 무게·부피 | 롱패딩 등 | "무게로 활동성 제약 가능" |
| 수축률 | 면100%·생지 | "세탁 후 수축 가능 — [시점] 재진단" |

기준옷 소재도 A축에 있으면 조회(안 물음), 없을 때만 폴백 질문.

## 8. 시점·재사용 게이트 (원칙 7)

- `worn_at = 세션 시각` 자동. 기간 안 물음.
- 카드 재사용은 **체중 불변**일 때만. 바뀌면 재사용 끊고 새 버전.
- 브랜드 실측 보정(A)엔 과거 입력도 유효(옷 치수는 체중 무관).

## 튜닝 파라미터 (Phase 0 캘리브레이션)

`δ`(존별 여유 스텝), `k`(암홀 요구 계수), `target_ease` 기본값, 병목 임계값. 시드 3브랜드 × 상·하의로 "신호 → 추천"이 서는지 검증하며 조정. 킬 메트릭(동의율 ≥65%)이 조정 기준.
