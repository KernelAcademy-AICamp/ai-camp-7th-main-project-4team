# 모바일 소비자 화면 — 핸드오프 노트

> **이 문서는 무엇?** 소비자(index·diag·result·card) 화면의 **모바일 UI 방향과 구현 스펙**을 정리한 핸드오프 노트입니다. 현재는 데스크톱 우선 반응형(브레이크포인트 900~560px, 뷰포트 메타 전 페이지)에 **모바일 네이티브 패턴**을 얹는 작업이에요. 마크업·CSS는 디자인(소희), 탭바 이벤트 배선은 개발이 맡습니다.

## 0. 결정 — 하단 내비 = **A (4탭 + 중앙 FAB 진단)**

내비 시안 4종 비교 후 **A** 채택. 핵심 결정축은 "진단(킬 액션)을 얼마나·어디에 노출하느냐".

| 시안 | 구성 | 트레이드오프 | 결과 |
|---|---|---|---|
| **A** | 4탭 + 중앙 FAB 진단 | 진단 진입률 최상 · FAB가 트렌디 | ✅ 채택 |
| B | 3탭(진단=홈 CTA) | 현행 최소 변경 · 진단 진입 한 단계 더 | |
| C | 4탭 균등(FAB 없이) | 절제 · 강조 약함 | |
| D | 5탭 + FAB(기록 추가) | 기능 확장형 · 5탭 폭 빡빡 | 나중 검토 |

시안(인터랙티브 목업): [상세 A 시안](https://claude.ai/code/artifact/61cb7788-0f07-4822-947f-0b2d1cf41350) · [내비 4종 비교](https://claude.ai/code/artifact/97801642-c1f8-4dbe-a60d-451bfc28a36e)

## 1. 브레이크포인트

- 기존 데스크톱 반응형은 **그대로 유지**. 모바일 셸은 `@media (max-width:640px)`에서 활성.
- 진단 화면은 diag.css 기존 `600px` 브레이크포인트와 정합되게.

## 2. 하단 탭바 (Bottom Tab Bar)

| 항목 | 스펙 |
|---|---|
| 위치 | `position:fixed; left:0; right:0; bottom:0` |
| 높이 | `56px + env(safe-area-inset-bottom)` (노치·홈바 대응) |
| 탭 | 홈 / **진단(중앙 FAB)** / 샵 / 마이 |
| FAB | 중앙 탭을 `margin-top:-18px` 띄운 `--green` 원형 + 흰 아이콘 |
| 활성/비활성 | `--green` / `--sub` |
| 콘텐츠 여백 | `body`(또는 셸)에 `padding-bottom`으로 탭바에 안 가리게 |
| 노출 | 홈·샵·마이에서만. **진단 플로우에선 숨김**(§4) |

## 3. 상단바 (모바일)

- 현재 `.navr`(스타일리스트 지원·알림벨·로그인·이름)가 좁은 폭에서 붐빔.
- 모바일에선 **로고 + 알림벨만** 남기고, 로그인·이름은 **마이 탭**으로 이동.

## 4. 진단 = 포커스 플로우 + 하단 고정 CTA

- 진단 진입 시 **탭바 숨김**(`display:none`) → 몰입.
- 하단에 **sticky CTA 바**: 진행률 바 + "다음". `env(safe-area-inset-bottom)` 패딩.
- 상단에 **뒤로가기**(이탈 경로) + 스텝 표시("2 / 5").
- **로직 불변** — `#nextbtn`·`footerAction()` 그대로, 감싸는 컨테이너만 CSS로 하단 고정.

## 5. 터치 · 입력

| 규칙 | 값 |
|---|---|
| 터치 타깃(세그먼트·칩·옵션) | `min-height:44px` |
| 눌림 피드백(hover 대체) | `:active { transform:scale(.96) }` 등 |
| 입력 폰트 | `font-size:16px` 이상 (iOS 포커스 자동 확대 방지) |
| 칩 레이아웃 | `flex-wrap` 줄바꿈 (B안 서비스·상황 칩 포함) |

## 6. JS ↔ DOM 계약 (배선)

- 탭 버튼 → 기존 `go('home' | 'shop' | 'my')` 그대로 재사용.
- 진단 FAB → 홈 CTA와 동일 진입(`diag-basic.html`).
- **디자이너가 탭바 마크업·`id`/`class` 정의 → 개발이 이벤트 연결.** (CLAUDE.md: DOM은 디자이너 정의, JS 참조 — 바꾸면 상호 알림)

## 7. 토큰 (하드코딩 금지 · `web/css/tokens.css`)

- 메인: `--green:#2E4A3B` · `--green-soft` · `--green-mid`
- 면/글: `--bg` · `--surface` · `--ink` · `--sub` · `--line`
- 기존 컴포넌트 재사용: `.chip` / `.chip.on` · `.btn` / `.btn.full`
