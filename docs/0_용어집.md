# 핏팅 용어집 (Glossary)

> **이 문서는 무엇?** 기획 문서 전체에 흩어져 있던 용어 풀이를 한곳에 모은 **정본 용어집**입니다. 각 문서는 이제 용어를 다시 풀지 않고 여기를 가리킵니다. 처음 읽는 분은 이 문서를 옆에 두고 1번(제품정의)부터 읽으세요.

---

## 1. 제품 · 전략

| 용어 | 뜻 |
|---|---|
| **킬 메트릭**(kill metric) | 이 하나가 안 나오면 서비스를 접거나 갈아엎는 '생사 기준' 지표. 핏팅은 **추천 사이즈 정확도 동의율 ≥ 65%**. |
| **North Star Metric**(북극성 지표) | 팀이 장기적으로 키우려는 단 하나의 핵심 숫자. 핏팅은 "주간 유료 추천 채택 수". |
| **Aha 모먼트** | 사용자가 "오, 좋다!"를 처음 느끼는 순간. 핏팅은 "안 입어본 브랜드인데 이 사이즈면 맞겠다"는 확신이 드는 순간. |
| **페르소나** | 대표 사용자 한 명을 구체적 인물로 그린 것. |
| **JTBD**(Jobs To Be Done) | 그 사람이 "이걸 왜 쓰는가"를 상황·목적으로 적은 문장. |
| **MVP**(Minimum Viable Product) | 최소한으로 만들어 가설을 검증하는 첫 버전. |
| **피벗**(pivot) | 사업 방향을 크게 바꾸는 것. 핏팅은 "전문 쇼퍼 연결(구 핏메이트)" → "사이즈 번역"으로 피벗함. |
| **페이크도어**(fake door) | 실제 기능은 안 만들고 버튼만 두어 "얼마나 누르나"로 수요를 떠보는 방법. |
| **위저드 오브 오즈**(wizard of oz) | 자동인 척하지만 뒤에서 사람이 손으로 처리하는 방법. |
| **전환신호**(conversion signal) | 사용자가 관심을 실제 행동으로 보이는 비율. 핏팅 보조지표 ≥ 25%. |
| **해자**(moat) | 경쟁자가 쉽게 못 따라오는 방어벽. 핏팅은 미표기 부위 검열추정 + 브랜드 교차 데이터. |
| **lazy 가입**(lazy registration) | 회원가입을 초반에 강요하지 않고, 가치를 느낀 뒤(저장·매칭 시점)로 미루는 것. |

## 2. 진단 · 엔진

| 용어 | 뜻 |
|---|---|
| **역산**(逆算) | 결과에서 원인을 거꾸로 계산하기. 착용 경험(옷이 어땠는지)에서 몸 치수·체형을 추정. |
| **여유**(ease) | 옷이 몸보다 얼마나 큰가(cm). `여유 = 옷 치수 − 몸 치수`. 엔진의 핵심 개념. |
| **A축 / B축** | A축 = 브랜드가 파는 옷의 실제 치수(물건 쪽). B축 = 착용경험으로 역산한 사용자 신체 치수(몸 쪽). |
| **핏라인**(FitLine) | 옷의 여유 스타일. skinny·slim·regular·loose·oversize 5단계. "이 옷이 의도한 여유량". |
| **기반 / 파생 카테고리** | 기반 = 상의+하의(실착으로 체형 확정). 파생 = 아우터·원피스·치마(선호핏만). |
| **앵커브랜드**(anchor brand) | 착용경험을 **물어볼 때** 기준으로 삼는 대표 브랜드 7종. 입력용 기준 목록이지 추천 결과가 아님. |
| **둘레 / 너비 / 길이**(Comparison) | 부위 재는 방식. 둘레=단면×2로 환산, 너비=그대로, 길이=여유개념 없음(기장감). |
| **δ(델타)** | "한 칸 조정폭". "가슴이 꼈다" → 가슴이 옷보다 δ만큼 크다. 실측으로 맞춰가는 튜닝값. |
| **EASE_BANDS**(느낌↔센티 환산표) | "끼임/딱맞음/여유/큼" 같은 말을 여유 cm 구간으로 바꾸는 표. 부위별로 폭이 다름. |
| **병목**(bottleneck) | 부위마다 요구가 엇갈릴 때 가장 크게 모자란 부위. 그 부위에 맞춰 사이즈를 정함. |
| **숨은 치수** | 맞춤옷은 재는데 기성복 사이즈표엔 없는 치수(겨드랑이·앞뒤품 등). 불편의 원천이자 해자 구간. |
| **병목 플래그**(PAIN_FLAGS) | 표에 잘 안 적히지만 불편을 만드는 부위(팔·목·종아리). 역산 대신 브랜드 치수 역추정에 씀. |
| **프로버넌스**(Provenance, 출처메타) | "이 숫자 어디서 언제 어떻게 가져왔나". 출처·날짜·방식·신뢰도. |
| **대리지표**(proxy) | 비공개 판매량 대신 쓰는 리뷰 수·인기순 등. "이건 대리지표"라고 표시. |
| **스냅샷** | 찍은 뒤 안 바뀌는 사진 같은 기록. 진단 결과는 시점별 스냅샷으로 저장(덮어쓰지 않음). |
| **8유형 / 핏 타입** | 결과 카드에 그리는 스타일링 체형 8종(STR/TRI/INV/HRG/BAL/DIA/RND/TUB). 표현용 캐릭터. |
| **아키타입**(archetypes) | 역산 결과를 맞대볼 실측 부위벡터 70종(표준38+비만32). 8유형 카드와 **다른 층**. |

## 3. 매칭 도메인 (쇼퍼찾기)

| 용어 | 뜻 |
|---|---|
| **마켓플레이스**(marketplace) | 파는 사람과 사는 사람을 연결하는 장터. 핏팅 매칭은 사용자·전문가·운영자 3면. |
| **핸드오프**(handoff) | 앞 단계 정보를 다음 단계로 그대로 넘기는 것. 진단 결과카드가 전문가 화면(E5)으로 넘어감. |
| **따뜻한 리드 / 차가운 리드**(warm/cold lead) | warm = 진단까지 마치고 온 전환 가능성 높은 사용자. cold = 진단 없이 바로 온 사용자. |
| **콜드스타트**(cold start) | 양쪽(수요·공급)이 다 없어 서비스가 안 도는 초기 상황. |
| **노쇼**(no-show) | 예약해 놓고 안 나타남. |
| **입찰(역경매)** | 사용자 요청에 전문가들이 가격·제안으로 입찰 → 사용자가 낙찰. 지명 요청과 병행. |

## 4. 디자인

| 용어 | 뜻 |
|---|---|
| **토큰**(token) | 색·크기 같은 디자인 값을 이름표로 정해둔 것(`--green` 등). 정본은 `web/css/tokens.css`. |
| **잉크블랙** | 새까만 검정 대신 살짝 부드러운 검정(`#16140F`). |
| **딥그린** | 핏팅의 단일 포인트색(`#2E4A3B`). 강조·활성에만 씀. |
| **요체** | 말끝을 `~해요·~예요`로 맺는 부드러운 문체. 핏팅 UX 라이팅 기본. |
| **핏(fit)** | 브랜드 언어. 핏 진단·핏 타입·핏 카드·핏 번역 등 핵심 개념에 일관되게 붙임. |

## 5. 거래 · 금액 (요청~정산)

> 금액 라벨은 **거래 단계마다 하나씩** 고정한다. **고객·쇼퍼 화면 동일 표기.** (2026-07-18 통일)

| 단계 | 정본 라벨 | 뜻 |
|---|---|---|
| **오픈 요청** | **예산** | 여러 스타일리스트에 뿌린 요청에 고객이 적은 예산 범위(예: 5~10만). |
| **지명 요청** | **예상 가격** | 특정 스타일리스트 지명 시 서비스 기준가로 안내되는 예상 금액(항상 "예상 가격", 축약 금지). |
| **수락 후** | **확정 금액** | 수락으로 확정된 거래 금액. |
| **상담중(쇼퍼→고객)** | **결제 예정 금액** | 입금 요청 전, 대화로 맞춰 안내할 금액(요청하면 잠금). |
| **결제창 내역** | **서비스 금액 · 수수료 · 결제 금액** | 결제 모달의 금액 분해(기본가·수수료·최종 결제액). |
| **정산** | **결제 금액** | 완료 확인 후 스타일리스트에게 지급되는 보관 결제금. |

- 통일 전 흔들리던 표기: ~~고객 예산~~ → **예산**, ~~안내할 금액~~ → **결제 예정 금액**, ~~예상 98,000원(축약)~~ → **예상 가격 98,000원**.
- **안전결제**(구 에스크로): 완료 확인 전까지 핏팅이 결제금을 보관하고, 확인 후 스타일리스트에게 정산하는 안전결제 방식. UI 라벨은 "안전결제"로 통일(내부 개념명은 에스크로).
