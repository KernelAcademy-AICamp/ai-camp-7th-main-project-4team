# 핏팅 — 쇼퍼 지원 포털 기능정의서

> **이 문서는 무엇?** 쇼퍼(공급자)가 쓰는 **쇼퍼 지원 포털**의 화면·기능·데이터를 정의합니다. 고객이 보낸 견적 요청을 받아 응답하고, 프로필·정산·후기를 관리하는 공급자 측 앱이에요.
> 관련: 가입 온보딩 상세는 [쇼퍼가입-화면정의서.md](쇼퍼가입-화면정의서.md), 쇼퍼찾기↔포털 데이터 통일은 [쇼퍼-데이터-계약.md](쇼퍼-데이터-계약.md), 색·말투는 [8_디자인가이드.md](8_디자인가이드.md).
> **담당:** 팀장 · 파일 = `web/pro.html`·`web/js/pro.js`·`web/pro-signup.*`·`web/pro-quote.*`

**표시 약속:** ★ 구현됨 · ◐ 최소/데모 · ○ v2(나중)

---

## 0. 개요

고객 앱(`index.html`)의 **"쇼퍼 지원"** 을 누르면 진입. 쇼퍼는 여기서 **① 가입 → ② 프로필 등록 → ③ 요청 응답 → ④ 정산·후기 관리** 를 한다.

```
고객 홈  ─[쇼퍼 지원]→  가입 온보딩(pro-signup)  ─완료→  쇼퍼 포털(pro.html)
                          (이미 가입 시 바로 포털)              │
                                                    요청 클릭 → 견적서(pro-quote)
```

- **저장:** 백엔드 없이 `localStorage` — 프로필 `fitting.pro.profile`, 요청 `fitting.pro.reqs` (§7)
- **페르소나:** "소희 쇼퍼"

## 1. 정보구조(IA)

```
쇼퍼 포털(pro.html)
 ├─ 헤더: 로고(→홈) · 알림 벨 · 계정(내프로필/로그아웃)
 └─ 사이드 메뉴(6)
     ├─ 📊 대시보드        (2.2)
     ├─ 🧾 요청 내역        (2.3)
     ├─ 🙋 먼저 제안        (2.4)
     ├─ 👤 내 프로필        (2.5)
     ├─ ⭐ 받은 후기        (2.6)
     └─ 💰 정산            (2.7)
 부속 화면
     ├─ 가입 온보딩 (pro-signup.html)   (2.1)
     └─ 견적서       (pro-quote.html)   (2.8)
```
> 사이드 메뉴는 **패널 전환**(SPA식) — 한 페이지 안에서 화면만 바뀜. 견적서·가입만 별도 페이지.

## 2. 화면별 기능정의

### 2.1 가입 온보딩 (pro-signup.html) ★
5단계 위저드. 자세한 화면정의는 [쇼퍼가입-화면정의서.md](쇼퍼가입-화면정의서.md).

| 단계 | 하는 일 |
|---|---|
| SP-0 소개 | 쇼퍼 역할 안내 → [쇼퍼 시작하기] |
| SP-1 가입 | 이메일·(연락처)·약관 동의 |
| SP-2 기본정보 | 활동명 · **서비스 유형별 가격**(온라인/동행쇼핑/이미지컨설팅) |
| SP-3 이력 | 소개 · 전문분야·스타일 |
| SP-4 포트폴리오 | 대표 작업 사진(최대 8) |
| SP-5 완료 | → 쇼퍼 포털 |

- **재방문:** 이미 가입(`registered:true`)이면 **화면 렌더 전** `<head>` 스크립트가 `pro.html`로 즉시 이동(깜빡임 없음). ★

### 2.2 대시보드 (#dash) ★
> 로그인 후 첫 화면 — "지금 상황"을 3초에 파악.

| 요소 | 내용 |
|---|---|
| 인사말 | "안녕하세요, {활동명}" (프로필 이름 반영) |
| 통계 4 | **이번 달 수익** · 신규 요청 · 진행 중 · **평점** |
| 지금 응답 필요 | 신규 요청 목록 + [상세 보기] → 견적서 |
| 최근 요청 | 최신 3건 |

- 수익 = 완료 건 금액 합계(실계산), 평점 = 받은 후기 평균(실계산). ★

### 2.3 요청 내역 (#inbox) ★
> 고객이 보낸 견적 요청을 상태별로 관리. (마이페이지 "요청 내역"과 UI 통일)

| 그룹(헤더 아이콘+건수) | 포함 상태 |
|---|---|
| 새 요청 | 신규 |
| 진행 중 | 수락됨 |
| 완료·취소된 요청 | 완료 · 거절 · 취소 |

- 각 행: 고객 이니셜 · 이름·상황 · **서비스 배지**(온라인/동행/이미지) · 체형·예산·날짜 · 상태 배지 · [›]
- **최신순 정렬**, 요청 클릭 → 견적서(2.8) ★

### 2.4 먼저 제안 (#propose) ★ (역방향 제안)
> 고객이 "제안받기"를 설정하면(고객 앱 영역·미구현) 그 목록이 뜨고, 쇼퍼가 **먼저 제안**.

| 요소 | 내용 |
|---|---|
| 제안받기 원하는 고객 | 후보 목록(데모) — 이름·상황·체형·예산·요청메모 + [제안하기] |
| [제안하기] | 인라인 폼(금액·메시지) → 발송 → "내가 보낸 제안"으로 이동, 후보 목록에서 제거 |
| 내가 보낸 제안 | 발송한 제안(제안발송) 목록 → 클릭 시 견적서(보낸 제안 모드) |

### 2.5 내 프로필 (#profile) ★
보기 ↔ 편집 토글. **데이터 계약([쇼퍼-데이터-계약.md](쇼퍼-데이터-계약.md))에 정합.**

| 항목 | 편집 | 저장 필드 |
|---|---|---|
| 프로필 사진 | 업로드·기본(SVG 캐릭터)·클릭 확대 | `avatar` |
| 활동명 | 텍스트 | `name` |
| 서비스별 가격 | 온라인/동행쇼핑/이미지컨설팅 토글+가격 | `services[{type,price,mode}]` |
| 활동 지역 | **대면 서비스(동행·이미지)** 켜면 노출, 지역 태그 | `services[].regions` |
| 소개 | 한 줄 텍스트 | `bio` |
| 전문 분야 | 7종 중 **최대 3**(상황: 소개팅·면접 등) | `occ[]` (코드) |
| 전문 스타일 | 8종 중 **최대 3**(캐주얼·미니멀 등) | `tags[]` |
| 포트폴리오 | 사진 + **착용 모델 키·몸무게** | `portfolio[{src,height,weight}]` |

- 저장 시 화면(헤더·사이드·프로필) 즉시 갱신. `mode`는 타입 고정(온라인→비대면 / 동행·이미지→대면). ★

### 2.6 받은 후기 (#reviews) ★
- 완료 건에 고객이 남긴 후기(별점+글) 목록
- 각 후기에 **쇼퍼 답글** 달기 ★

### 2.7 정산 (#settle) ★
- 상단: **이번 달 수익 · 완료 건 · 건당 평균**
- 내역 리스트: 고객·서비스·날짜·금액·정산상태(대기)
- 완료 건 × 금액 실계산 (실제 정산연동은 v2 ○)

### 2.8 견적서 (pro-quote.html) ★
> 요청/제안 클릭 시 진입하는 상세 페이지.

| 영역 | 내용 |
|---|---|
| 헤더 | 로고(→홈) · ← 요청 내역으로 |
| 좌 | **고객 체형진단 결과 상세** — 세그먼트 측정(상체/하체/취향, 마이페이지와 동일 디자인). 브랜드 추천 제외 |
| 우 | 요청 내용(서비스·상황·예산·일정·한줄) + **제안 액션** |
| 액션(상태별) | 신규→[수락]/[거절] · 진행중→[완료 처리]/[취소 처리] · 거절/취소→[되돌리기] · 보낸제안→[고객 수락·데모] |
| 거절·취소 | **사유 입력** + 되돌리기(복구) ★ |

- 보낸 제안(dir:out)은 "제안 대상 + 현황"으로 표시.

### 2.9 헤더 (공통) ★
- **로고** 클릭 → 고객 홈(index.html)
- **알림 벨** → 신규 건수 배지, 클릭 시 요청 내역
- **계정 메뉴**(이름 클릭) → 내 프로필 / 로그아웃
- 아이콘은 마이페이지와 동일한 **선(line) SVG** 스타일

## 3. 요청 라이프사이클 (상태 머신)

```
받은 요청:  신규 ──[수락]──▶ 수락됨 ──[완료 처리]──▶ 완료 ──▶ (고객 후기)
             │                  │
          [거절]             [취소 처리]
             ▼                  ▼
           거절 ◀[되돌리기]    취소 ◀[되돌리기]

보낸 제안:  제안발송 ──[고객 수락·데모]──▶ 수락됨 ─(이하 동일)
```
- 상태 배지 색: 신규(초록 채움) · 수락됨/완료(초록 연함) · 거절/취소(회색)

## 4. 데이터 계약 / 저장 (localStorage)

### 4.1 프로필 — `fitting.pro.profile`
```js
{
  registered, name, avatar, bio,
  services: [
    { type:'online',   price, mode:'비대면' },
    { type:'shopping', price, mode:'대면', regions:['서울 강남', …] },
    { type:'image',    price, mode:'대면', regions:[…] }
  ],
  occ:  ['date','daily', …],     // 전문분야=상황 코드(최대 3) → 쇼퍼찾기 occ[]
  tags: ['미니멀','시크', …],     // 스타일(최대 3) → 쇼퍼찾기 tags[]
  portfolio: [{ src, height, weight }, …]   // 사진 + 착용 모델 스펙
}
```
> 쇼퍼찾기(`index.js` EX)와 **필드 통일** — 포털 입력이 쇼퍼찾기 카드·필터로 흐르도록. 상세는 [쇼퍼-데이터-계약.md](쇼퍼-데이터-계약.md).
> ⏳ 남은 합의: 프로필 사진 매핑(포털 `avatar` ↔ 쇼퍼찾기 `photo`).

### 4.2 요청 — `fitting.pro.reqs`
```js
{ cust, occ, service:'online|shopping|image', budget, date,
  type, bodytype, cm, kg,           // 고객 체형(견적서 측정 표시용)
  status:'신규|수락됨|완료|거절|취소|제안발송',
  dir:'out'?,                        // 쇼퍼→고객 제안이면 out
  offer:{price,msg}?, review:{rating,text}?, reason? }
```

## 5. 데모 한계 · 다음(v2)

| 항목 | 지금(데모) | v2 |
|---|---|---|
| 저장 | localStorage(브라우저=쇼퍼 1명) | 백엔드 DB(쇼퍼 다수) |
| 고객↔쇼퍼 요청 | 각자 더미 데이터 | 같은 요청 레코드 공유(실연동) |
| 정산 | 금액 합계 표시 | 실제 정산·수익 관리 |
| 포트폴리오 사진 | 로컬 미리보기(dataURL) | 실제 업로드 |
| 프로필 사진 매핑 | avatar만 | 쇼퍼찾기 photo와 연동 |
