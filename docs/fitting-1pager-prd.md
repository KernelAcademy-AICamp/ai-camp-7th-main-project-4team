# 핏팅(Fitting) — 1-Pager PRD

## 1. 한 줄 요약
새 브랜드 옷을 사고 싶어도 "내 사이즈가 맞을지" 몰라 주저앉는 사람에게, 평소 입던 옷의 착용 경험을 기준으로 **브랜드를 가로질러 내 사이즈를 번역해주는** AI 진단 카드. 진단으로 사용자·사이즈 데이터를 모아, 그 위에 스타일리스트 매칭을 얹어 확장한다.

## 2. 문제 정의
새 브랜드로 건너가려는 사람은 "이 브랜드에서 내 사이즈가 뭔지"를 몰라 망설인다. 무신사 등은 이미 사이즈 추천을 하지만 *자기 몰 안 + 구매 이력*에 갇혀 있어, 새 몰·새 브랜드로 넘어가는 순간엔 아무도 번역해주지 않는다. 결국 실패가 두려워 늘 사던 브랜드의 비슷한 옷만 다시 사게 된다.

> 차별점이자 해자: ① 몰 교차 ② 구매 이력 불필요(착용경험 기반) ③ 중립성 — 직접 팔지 않으니 건너가게 도울 동기가 있고, 몰을 가로지르는 사이즈 데이터는 개별 플랫폼이 구조적으로 못 만든다(무신사는 자기 울타리 안에 갇힘).
> 설문 근거: 새 옷 구매 시 막힘 — 스타일 40% / 사이즈 38%. 스타일은 전문가가, 사이즈는 도구가 푼다.

## 4. North Star Metric
- 주간 유료 추천 채택 수.
- 본 MVP는 그 입구인 **사이즈 번역 정확도 + 유료 전환신호**를 검증.

## 5. Aha Moment
- 진단 카드에서 *"평소 입던 옷 기준으로, 한 번도 안 입어본 브랜드인데 이 사이즈면 맞겠다"*는 확신이 처음 드는 순간.

## 6. MVP 기능 (2개)
- **AI 체형·사이즈 진단 [LLM + 규칙]** — 입력: 성별·키·몸무게·나이 + *입어본 옷 착용경험 보정* + 선호핏. 출력 카드: ① 체형 프로필(캐릭터) ② 핏 인사이트 ③ **주요 브랜드별 추천 사이즈**.
- **전문가 추천 받기 (페이크도어)** — 클릭만 로깅해 유료 전환신호 측정(실매칭 X).

## 7. 단계적 범위 (Now / Next)
- **Now(MVP):** 진단으로 사용자·사이즈 데이터 확보(주요 브랜드 N개 한정).
- **v2:** 전문가 실매칭으로 수익화 — 매칭 수수료(B2C)·커머스 제휴 커미션.
- **v3:** 축적 데이터를 사이즈 API(B2B, 쇼핑몰 반품 절감)로 확장 + 상황별(소개팅·면접·결혼식·여행 등) 카테고리·오프라인·퍼스널컬러. 상황은 페인·지불 의향이 집중되는 지점이라 매칭의 과금 단위가 된다.
- *"데이터 확보 → 수익화 → 데이터 확장"의 사업 단계.*

## 8. 성공 기준
- ★ **킬 메트릭** — 추천 사이즈가 "내가 실제 입는 사이즈와 맞다" 동의율 **≥ 65%** (아는 브랜드로 검증 → 실구매 없이 엔진 정확도 확인. 미달 시 엔진 교체/중단).
- 보조: "안 사본 브랜드도 사보겠다" 의향 ≥ 50%(페인 해소) / 유료 클릭률 ≥ 25%(전환신호).

## 9. 열린 질문
- 체형·사이즈 추정 로직(규칙 vs LLM vs 혼합) — *이 MVP가 킬 메트릭으로 답할 것.*
- 착용경험 보정 몇 벌·어떤 부위.
- 브랜드 실측 확보·정규화(차별점이자 최대 난관).
- 페이크도어 클릭의 허수 보정.
