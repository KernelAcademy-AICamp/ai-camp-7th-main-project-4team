# 사이즈코리아 정규화 시드 (B축 = 인체 데이터)

원본(`data/body/raw/*.xlsx`)을 엔진이 바로 읽도록 정규화한 JSON. **단위 cm/kg 통일, 999.9 등 결측코드 제거.**
출처: 사이즈코리아 제8차 인체치수조사(2020~2024) / 국가기술표준원. 사용 시 출처표기 의무.

생성 스크립트는 일회성(`scratchpad`)이며, raw 엑셀이 갱신되면 재생성한다.

## 파일

### `body-base-model.json` — 기초신체 추정기 (0벌 디폴트)
성별별 회귀계수. 착용경험이 없을 때 `(성별·키·몸무게·나이) → 부위 추정`.
```
part_cm = a_height·키(cm) + b_weight·몸무게(kg) + c_age·나이 + intercept
```
- 부위: 젖가슴/가슴(윗)/허리/엉덩이둘레, 어깨가쪽사이길이, 팔길이, 위팔둘레. `bodyDimension`이 엔진 스키마 키.
- 각 부위에 `r2`·`rmse_cm`(불확실성)·`n` 동반. **둘레는 몸무게 지배(R²≈0.8), 어깨는 키 지배(R²≈0.4).**
- 나이는 특히 **허리**에서 유효(+1.7cm/10년).

### `body-distribution.json` — 분포 (신뢰도·위치 표시용)
성별 × 부위(+키·몸무게): mean·sd·p5·p50·p95. 추정치가 모집단 어디인지(P##) 표기에 사용.

### `archetypes.json` — 체형 캐릭터 라벨 (숫자 아님)
- `standard`: 표준체형 38개(성별·연령대·체형명 + 부위벡터 cm).
- `obese`: 비만체형 32개(이름·**설명문** 포함 — LLM 캐릭터 서술 grounding).
- 용도: **역산 후** 사용자 부위벡터를 nearest-archetype로 스냅 → 결정론적 체형 라벨. 0벌 단계 분류엔 부적합(체형들이 키·몸무게로 안 갈림).

## 주의 (known caveats)
- 체형명("역삼각/삼각/사각")은 어깨vs엉덩이 실루엣이 아니라 **드롭·부피·신장체중 비례** 분류 — 어깨/엉덩이 비율은 38개 전부 0.37~0.43로 거의 일정. 곧이곧대로 매핑 금지.
- `obese`의 `겨드랑둘레`는 원본 값 크기(≈12cm)가 둘레로 보기엔 작아 **항목 라벨 모호**(겨드랑두께 가능성). 캐릭터 보조로만, 수치 매칭에 쓰지 말 것.
- 몸무게 P95≈94kg(남)/75kg(여). 그 바깥(고도비만)은 **외삽**이라 추정 신뢰도 하락 → `obese` 아키타입으로 보완.
- 이 모델은 **0벌 디폴트(부위당 ±2~3.5cm)**. 착용경험 역산이 그 위를 덮어써 ±2cm로 좁히는 게 엔진 목표.
