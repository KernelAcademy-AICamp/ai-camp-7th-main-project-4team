## 3. 8유형 데이터 — 단일 출처(Single Source) 확정

### 정본
**`진단카드_8유형.json`** (v1.0, 2026-06-29) = 유일한 출처. 완비 상태:
- 8유형 각: `code · name · sizeKorea · silhouette · point(색) · profile[2줄] · fitOk[2] · fitNo · insight · signature[3] · match(궁합)`
- `silhouetteClipPaths`: 8개 실루엣 polygon/ellipse 좌표
- `axes`: primary(8 archetype) · secondaryTag(frame L/C) · gender(머리만 다름, 추천분기 v2)

### 현재 중복 (제거 대상)
| 파일 | 중복 내용 | 조치(④) |
|---|---|---|
| `결과카드_단독.html` | `T` 객체 + clipPaths 인라인 하드코딩 | JSON을 import/fetch로 렌더 |
| `체형진단_결과웹.html` | 동일 데이터 재하드코딩 | 위와 동일 출처 참조 |
| `index.html` | (유일하게) JSON fetch 중 | 참조 방식 정본 |
| `_보관/진단카드_공유목업.html` | **구 6유형** 좌표계 | 폐기 |

### 규칙
1. 8유형의 이름·색·프로필·핏·궁합·실루엣은 **오직 JSON에서** 읽는다. 화면 코드에 재기입 금지.
2. ④에서 JSON을 공용 위치로 이동(예: `_통합/data/bodytypes.json` 또는 서비스 `data/`), 모든 렌더러가 한 곳 참조.
3. **엔진 연결점(seam):** 이 JSON은 "표현"(카드용). sangmin 엔진의 upper×lower 역산 결과 → `code` 매핑은 `서비스구조-뼈대.md §6`의 별도 매핑 테이블이 담당(Phase C 전).

### 갭 / 후속
- gender별 **추천 분기** 없음(현재 캐릭터 머리색만 분기) → v2.
- `sizeKorea`·`frame` 은 근거 태그(카드 노출 X, 문서/매핑용).

## 4. ④(서비스 기반 생성)에서 할 일 — 이 구조화의 실행
1. 공용 스캐폴드에 `tokens.css` + 폰트(Pretendard·SUIT) 로드 1곳.
2. `진단카드_8유형.json` 공용 위치로 이동, 렌더러 중복 제거(§3).
3. §2 컴포넌트를 뼈대 노드(§1)에 맞춰 배치 + 토큰화(색·폰트 변수 치환, 블루→딥그린).
4. 결과 카드 예외 규칙 보존(다크+8유형색), 나머지 화면 딥그린 통일.
5. sohee 위저드 셸은 자리만 잡고, 입력 내용은 sangmin 모듈이 들어올 자리로 비워둠(⑤⑥ 연결점).
