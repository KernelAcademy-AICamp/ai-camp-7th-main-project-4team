# 디자인 언어 (정본) — fitting 딥그린

> Phase A ②: sohee 디자인을 통합 정본으로 정립.
> 소스: `sohee/디자인가이드.html`(딥그린, 2026-07) + `05_스타일가이드.md`(형태 토큰) + `UX라이팅가이드.html`.
> 구현: **`_통합/tokens.css`** (실제 CSS 변수·컴포넌트 참조). 모든 화면은 이 토큰만 참조한다.

---

## 1. 철학 한 줄
**오프화이트 베이스 + 딥그린 포인트 1색 + 잉크블랙 다크.** 차분하고 신뢰감. 화려한 UI보다 **번역 정확도**가 주인공이라, 디자인은 절제로 그 신뢰를 받친다. (배경색으로 위계를 만들지 않는다 — "AI 티" 억제)

## 2. 5원칙 (되돌리지 말 것)
1. **색은 딥그린 하나.** 구조(배경·텍스트·선)는 뉴트럴, 강조는 딥그린만. 여러 색 금지.
2. **버튼은 검정(잉크블랙).** CTA는 `--ink`. 초록은 '포인트'로만 → 색이 흔해지지 않게.
3. **폰트는 Pretendard 하나.** 세리프·별도 서체 없이 **굵기·자간·여백**으로만 위계. (예외: 측정 수치=SUIT, §5)
4. **넓고 여유 있게.** 화면 크게, 간격 넉넉히. 결과는 세로로 길게 말고 **좌우 2단**.
5. **결과 카드는 예외.** 체형진단 카드만 **다크 배경 + 8유형별 포인트색**. UI 딥그린과 별개로 유지 — **절대 재스킨하지 않음.**

## 3. 색 시스템

### 뉴트럴 베이스
| 이름 | 값 | 용도 |
|---|---|---|
| Paper `--bg` | `#F6F6F5` | 기본 배경(오프화이트) |
| Surface `--surface` | `#FFFFFF` | 카드·표면 |
| Ink `--ink` | `#16140F` | 본문·버튼(순검정 대신) |
| Ink-soft `--ink-soft` | `#3A382F` | 진한 보조 |
| Sub `--sub` | `#6E6B63` | 보조 텍스트 |
| Sub2 `--sub2` | `#A6A29A` | 옅은 텍스트·캡션 |
| Line `--line` | `#E6E5E1` | 경계선·카드 테두리(1px) |
| Line2 `--line2` | `#DAD8D2` | 입력·옵션 테두리(1.5px) |
| Soft `--soft` | `#EDEDEA` | 태그·연한 면 |

### 딥그린 액센트 (유일한 포인트색)
| 이름 | 값 | 용도 |
|---|---|---|
| `--green` | `#2E4A3B` | 강조 키워드·활성 탭/칩·핵심 |
| `--green-mid` | `#5A7A68` | 보조 강조 |
| `--green-light` | `#9CC1AE` | 다크 배경 위 강조 텍스트 |
| `--green-soft` | `#E7EFEA` | 활성 메뉴·뱃지 배경 |

### 기능색 (절제)
- `--warn` `#B0553F` — 경고·주의, 검증 **'틀림'(❌)**.
- **'맞음'(✅) = `--green` 겸용**, **'애매'(➖) = `--sub`**. (별도 성공 그린 안 씀 — 브랜드색과 안 섞음)

### 색 사용 규칙 (어디에 무엇)
| 요소 | 색 |
|---|---|
| 주요 버튼(CTA)·선택 탭/칩 | **잉크블랙** `--ink` (선택 칩은 딥그린 채움도 가능) |
| 강조 키워드·활성 상태·매칭도/정확도 뱃지 | **딥그린** `--green` / `--green-soft` |
| 로고·제목·본문 | `--ink` / `--sub` |
| 경고·검증 '틀림' | `--warn` |
> 한 화면에서 포인트색이 보이는 지점은 **의미 있는 곳 소수로.** 남발 금지.

## 4. 결과 카드 팔레트 (예외 · 데이터 주도)
결과 카드는 `--rc-bg #16171b` + 8유형색 1개(실루엣·코드·궁합·아이콘·상단 글로우에만).
`STR #9DB8FF · TRI #76D6C6 · INV #C2C7D0 · HRG #CF9BFF · BAL #8FD6A8 · DIA #FF9EC5 · RND #FF9D7A · TUB #B6A8FF`
→ 값은 `진단카드_8유형.json`을 single source로. (§Phase C에서 하드코딩 중복 제거)

## 5. 타이포
- **본문·한글 = Pretendard**(fallback Apple SD Gothic Neo). **측정 수치 = SUIT**(`--num`, tabular-nums) — cm·사이즈·매칭도% 등 "정밀=엔진 정체성"을 시각화. 둘 다 한글 지원(필수 규칙).
- 위계: Display 44/800 · H1 37/800 · H2 24/800 · H3 19/800 · 본문 16/450 · 보조 13.5/450 · 라벨 12/800(uppercase, +.10em).
- 자간: 본문 −.02em · 큰 제목 −.05em · 라벨 +.10em · 코드(STR·HRG) +.20em.
- 굵기 대비(800↔300)로 위계. 색으로 강조하지 않음.

## 6. 형태 토큰
- 라운드: 버튼/입력 12 · 칩/뱃지 20 · 카드 18 · 모달 24 · 결과카드 24(20~30).
- 테두리: 카드/구분선 `1px solid --line` · 입력/옵션 `1.5px solid --line2`.
- 그림자: 카드 hover `0 14px 34px rgba(20,22,28,.09)` · 드로어 `-22px 0 60px rgba(0,0,0,.18)`.
- 컨테이너: `--maxw 1240px`, 좌우 패딩 26px. 카드 hover = `translateY(-3px)`.

## 7. 컴포넌트 빠른 참조
| 컴포넌트 | 기본 | 선택/활성 |
|---|---|---|
| 버튼(주요) | 검정 채움·흰 글자 | hover `#000` / 비활성 `--line2` |
| 버튼(보조) | 흰 배경·1.5px 테두리 | hover `--soft` |
| 칩(필터) | 흰 배경·회색 | **딥그린 채움**·흰 글자 |
| 탭 | 회색 텍스트 | **딥그린** 텍스트 + 딥그린 밑줄 |
| 옵션 카드 | 테두리 `--line2` | 딥그린 테두리 + `--green-soft` |
| 토글 | 회색 | **딥그린** ON |
| 매칭도/정확도 뱃지 | — | **딥그린 pill**(`--green-soft`/`--green`) |
> ⚠️ 구 규칙에서 토글/뱃지/추천값이 **블루(`--ac`)**였음 → 전부 **딥그린**으로 치환.

## 8. UX 라이팅
- 보이스: "친절한 스타일 친구" — 친근·명확·신뢰.
- 기본 **요체(~해요)**. 브랜드 언어 = **'핏(fit)'**.
- **문장 속 이모지 지양.** 전문 패션용어 금지(테일러드·박시·로우라이즈 → 쉬운 말).
- 결과카드 '잘맞/피할 FIT'은 누구나 아는 말, 각 1줄. 방향성 팁은 'FIT INSIGHT'로.
